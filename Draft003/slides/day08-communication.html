📊 의사소통의 중요성

PM 업무의 90%는 의사소통!

프로젝트 실패 원인 중 30% 이상이 부적절한 의사소통에서 비롯됩니다.

의사소통 채널 수 공식

채널 수 = n × (n - 1) / 2
n = 이해관계자 수
5명
10 채널
10명
45 채널
20명
190 채널
50명
1,225 채널

인원이 늘어나면 채널은 기하급수적으로 증가! → 의사소통 관리 계획이 필수

📡 의사소통 방법

🏃 Push (밀어내기)
발신자가 정보를 보냄
수신 확인 X
이메일, 보고서, 메모
🔍 Pull (끌어오기)
수신자가 스스로 찾음
대규모 정보에 적합
인트라넷, Wiki, 공유 폴더
🤝 Interactive (대화형)
양방향 실시간 소통
가장 효율적
회의, 전화, 화상통화
의사소통 모델

발신자 → 인코딩 → 메시지/매체 → 디코딩 → 수신자 → 피드백
중간에 노이즈(잡음)가 메시지를 왜곡할 수 있습니다.

📝 회의 관리 Best Practice

효과적인 회의를 위한 원칙

  • 📋 아젠다 사전 배포 (24시간 전)
  • ⏰ 시작/종료 시간 준수
  • 👤 필수 참석자만 초대
  • 📝 회의록(MoM) 작성 및 공유
  • ✅ Action Item 명확히 (담당자 + 기한)
  • ⏲️ Timeboxing 적용

RAG 상태 보고 (Red-Amber-Green)

🔴 Red
심각한 문제. 에스컬레이션 필요
🟡 Amber
주의 필요. 리스크 존재
🟢 Green
정상 진행. 이슈 없음

📊 주요 보고서 유형

보고서 빈도 주요 내용 대상
주간 보고서 주 1회 이번 주 성과, 다음 주 계획, 이슈 PM → 스폰서/팀
월간 보고서 월 1회 전체 진척률, EVM, 리스크 PM → 경영진
이슈 보고서 수시 이슈 상세, 영향, 대응 PM → 관련자
변경 요청서 수시 변경 내용, 영향 분석 PM → CCB
종료 보고서 종료 시 프로젝트 성과 요약 PM → 전체
📋 실습: 주간 보고서 작성

인사관리 시스템 프로젝트의 4주차 주간 보고서를 RAG 상태와 함께 작성하세요.

## 📚 강의 구성

### 오전 세션 (4시간)

#### 1교시: 범위 관리 개요 (1시간)
- **범위(Scope)의 정의**
  - 제품 범위(Product Scope): 제품/서비스의 특성과 기능
  - 프로젝트 범위(Project Scope): 제품 범위를 인도하기 위해 수행해야 할 작업
- **범위 관리의 6개 프로세스**
  1. 범위 관리 계획 수립
  2. 요구사항 수집
  3. 범위 정의
  4. WBS 작성
  5. 범위 확인
  6. 범위 통제
- **범위 관리 계획서**: 범위를 정의, 검증, 통제하는 방법을 기술

#### 2교시: 요구사항 수집 (1.5시간)
- **요구사항의 분류**
  - 비즈니스 요구사항
  - 이해관계자 요구사항
  - 솔루션 요구사항 (기능적 / 비기능적)
  - 전환 요구사항
  - 프로젝트 요구사항
- **요구사항 수집 도구 및 기법**
  - 인터뷰(Interview): 1:1 또는 그룹 인터뷰
  - 포커스 그룹(Focus Group): 사전 선별된 이해관계자 그룹
  - 워크숍(Facilitated Workshop): JAD(Joint Application Design)
  - 설문조사(Questionnaire/Survey)
  - 벤치마킹(Benchmarking)
  - 브레인스토밍(Brainstorming)
  - 프로토타입(Prototype): 시각적 모형 제공하여 피드백 수집
  - 관찰(Observation) / Job Shadowing
- **요구사항 추적 매트릭스(RTM: Requirements Traceability Matrix)**
  - 각 요구사항의 출처, 상태, 검증 방법을 추적

#### 3교시: 범위 정의 및 프로젝트 범위 기술서 (1.5시간)
- **범위 정의**: 프로젝트 및 제품에 대한 상세한 설명을 작성
- **프로젝트 범위 기술서(Project Scope Statement) 구성요소**
  - 제품 범위 설명
  - 인도물(Deliverables)
  - 인수 기준(Acceptance Criteria)
  - 프로젝트 제외사항(Exclusions)
  - 제약조건(Constraints)
  - 가정(Assumptions)
- **제외사항 명시의 중요성**: "하지 않을 것"을 명확히 해야 범위 크리프 방지

### 오후 세션 (4시간)

#### 4교시: WBS 작성 (2시간)
- **WBS(Work Breakdown Structure)란?**
  - 프로젝트 인도물과 작업을 더 작고 관리 가능한 구성요소로 분해
  - 계층적 구조 (트리 형태)
- **WBS의 핵심 개념**
  - 작업 패키지(Work Package): WBS의 최하위 수준
  - 100% 규칙: 모든 작업이 포함되어야 함 (빠짐없이, 중복없이)
  - WBS 사전(WBS Dictionary): 각 구성요소에 대한 상세 설명
- **WBS 작성 방법**
  - 상향식(Bottom-up) vs 하향식(Top-down)
  - 분해(Decomposition) 기법
  - 인도물 기반 vs 단계 기반 WBS
- **WBS 작성 예시**
  - IT 시스템 구축 프로젝트 WBS 예시
  - 모바일 앱 개발 프로젝트 WBS 예시
- **WBS 사전 작성 요소**
  - 코드 번호, 작업 설명, 책임자, 일정, 원가, 인수 기준

#### 5교시: 범위 확인 및 통제 (1시간)
- **범위 확인(Validate Scope)**
  - 완성된 인도물의 공식적 인수
  - 고객/스폰서의 검토 및 승인
  - 범위 확인 vs 품질 통제: 인수 여부(범위) vs 품질 기준 충족(품질)
- **범위 통제(Control Scope)**
  - 범위 기준선 대비 모니터링
  - 범위 변경 관리
  - **범위 크리프(Scope Creep)**: 통제되지 않은 범위 확장
    - 원인: 불명확한 요구사항, 이해관계자 압력, 부적절한 변경 통제
    - 예방: 명확한 범위 기술서, 변경 통제 프로세스, 이해관계자 교육

#### 6교시: 🔨 실습 - WBS 작성 (1시간)
- **실습 시나리오**: 사내 인사관리 시스템 구축 프로젝트
- **실습 과제**
  - WBS 3단계 이상 작성
  - 각 작업 패키지에 대한 WBS 사전 작성 (최소 5개)
  - 요구사항 추적 매트릭스(RTM) 간이 작성
- 팀별 발표 및 상호 피드백

## 📝 핵심 키워드

| 키워드 | 설명 |
|--------|------|
| WBS | 작업분류체계, 프로젝트 작업을 계층적으로 분해 |
| 작업 패키지 | WBS의 최하위 수준 구성요소 |
| 100% 규칙 | WBS는 프로젝트의 모든 범위를 포함해야 함 |
| 범위 크리프 | 통제되지 않은 범위의 지속적 확장 |
| RTM | 요구사항 추적 매트릭스 |
| 인수 기준 | 인도물을 수락하기 위한 조건 |

## 📚 강의 구성

### 오전 세션 (4시간)

#### 1교시: 의사소통 관리 개요 (1시간)
- **의사소통의 중요성**: PM의 시간 중 약 90%가 의사소통
- **의사소통 관리의 3개 프로세스**
  1. 의사소통 관리 계획 수립
  2. 의사소통 관리
  3. 의사소통 감시
- **의사소통 채널 수 공식**
  - 채널 수 = n(n-1)/2 (n = 이해관계자 수)
  - 예: 10명 → 45개 채널, 15명 → 105개 채널
  - 1명 추가 시 채널 급증 → 의사소통 복잡도 증가
- **의사소통의 5C**
  - Correct(정확), Concise(간결), Clear(명확), Coherent(일관), Courteous(정중)

#### 2교시: 의사소통 모델 및 방법 (1.5시간)
- **의사소통 모델**
  - 발신자(Sender) → 부호화(Encode) → 메시지(Message) → 매체(Medium) → 해독(Decode) → 수신자(Receiver)
  - 잡음(Noise): 의사소통을 방해하는 요소
  - 피드백(Feedback): 메시지가 올바르게 전달되었는지 확인
- **의사소통 유형**
  - **공식적(Formal) vs 비공식적(Informal)**
    - 공식: 보고서, 공문, 프레젠테이션
    - 비공식: 이메일, 메신저, 대화
  - **문서(Written) vs 구두(Verbal)**
    - 복잡한 내용 → 문서
    - 긴급한 내용 → 구두
  - **수직적(Vertical) vs 수평적(Horizontal)**
- **의사소통 방법**
  - 상호작용(Interactive): 회의, 전화, 화상회의
  - 푸시(Push): 이메일, 보고서 배포, 메모
  - 풀(Pull): 공유 드라이브, 인트라넷, 지식 저장소

#### 3교시: 의사소통 관리 계획 수립 (1.5시간)
- **의사소통 관리 계획서 구성요소**
  - 정보 수요 분석 (누가, 무엇을, 언제, 어떻게)
  - 이해관계자별 의사소통 요구사항
  - 보고 주기 및 형식
  - 에스컬레이션 절차
  - 용어 사전(Glossary)
- **의사소통 기술 (Technology)**
  - 긴급성, 가용성, 보안, 민감도에 따라 선택
  - 프로젝트 규모와 분산 정도 고려
- **의사소통 매트릭스 작성**

### 오후 세션 (4시간)

#### 4교시: 의사소통 관리 실행 (1.5시간)
- **효과적 의사소통의 원칙**
  - 능동적 경청(Active Listening)
  - 비언어적 의사소통(Non-verbal): 표정, 자세, 제스처
  - 파라 언어(Paralingual): 톤, 속도, 강세
  - 메시지의 55-38-7 법칙 (바디 랭귀지-목소리-단어)
- **효과적 회의 운영**
  - 회의 전: 안건(Agenda) 사전 배포, 목적 명확화
  - 회의 중: 타임키핑, 참여 유도, 기록
  - 회의 후: 회의록(Minutes) 공유, 액션아이템 추적
  - 불필요한 회의 최소화

#### 5교시: 프로젝트 보고 체계 (1.5시간)
- **보고서 유형**
  - 상태 보고서(Status Report): 현재 시점의 프로젝트 상태
  - 진행 보고서(Progress Report): 기간별 성과 비교
  - 추세 보고서(Trend Report): 시간 경과에 따른 추세
  - 예측 보고서(Forecasting Report): 향후 성과 예측
  - 편차 보고서(Variance Report): 계획 대비 실적 차이
- **작업 성과 데이터 → 작업 성과 정보 → 작업 성과 보고서**
  - 데이터: 가공 전 원시 데이터 (예: 완료율, 지출액)
  - 정보: 분석된 데이터 (예: SPI, CPI)
  - 보고서: 의사결정을 위한 종합 문서
- **대시보드 및 시각적 보고**
  - 신호등 보고(RAG: Red-Amber-Green)
  - 번다운 차트(Burndown Chart)
  - 대시보드 구성 원칙

#### 6교시: 🔨 실습 - 의사소통 계획 및 보고서 작성 (1시간)
- **실습 1: 의사소통 매트릭스 작성**
  - 이해관계자 목록 기반 의사소통 매트릭스 작성
  - 각 이해관계자의 정보 요구, 주기, 방법 정의
- **실습 2: 주간 상태 보고서 작성**
  - 가상 프로젝트 데이터 기반 상태 보고서 작성
  - RAG 상태 표시 포함
- 팀별 발표 및 피드백

Information Security for PM

🔒 정보보안 3대 원칙 (CIA Triad)

🔐
기밀성 (Confidentiality)
인가된 사용자만 정보에 접근

대책: 암호화, 접근 통제,
최소 권한 원칙
✅
무결성 (Integrity)
정보가 부적절하게 변경되지 않음

대책: 해시, 디지털 서명,
변경 감사 추적
⚡
가용성 (Availability)
필요할 때 언제든 접근 가능

대책: 이중화, 백업,
DDoS 방어
PM과 보안의 관계

보안 요구사항은 프로젝트 범위·일정·비용에 직접 영향! 초기부터 보안을 고려해야 합니다 (Security by Design)

🛡️ 접근 통제

인증 (Authentication)

"누구인가?" 를 확인

인증 3요소
지식: 비밀번호, PIN
소유: OTP, 보안카드
행위/생체: 지문, 얼굴

MFA(다중 요소 인증): 2개 이상 요소 조합 → 보안 강화

인가 (Authorization)

"무엇을 할 수 있는가"를 결정

RBAC (역할 기반 접근 통제)
역할에 따라 권한 부여
예: 관리자, 편집자, 뷰어
최소 권한 원칙
업무에 필요한 최소한의 권한만 부여

⚠️ OWASP Top 10 (주요 보안 취약점)

# 취약점 설명 PM 대응
1 접근 제어 실패 인가되지 않은 리소스 접근 권한 체계 검증 요구
2 암호화 실패 민감 데이터 평문 전송/저장 암호화 정책 수립
3 인젝션 SQL Injection 등 악성 입력 보안 코딩 기준 적용
4 불안전한 설계 설계 단계의 보안 결함 위협 모델링 수행
5 보안 설정 오류 기본값 사용, 불필요 기능 노출 보안 설정 체크리스트
SQL Injection 예시

입력: ' OR '1'='1'; DROP TABLE users; --
→ DB의 모든 사용자 정보가 삭제될 수 있음! 입력값 검증(Validation)이 필수

📋 ISMS & 컴플라이언스

ISMS (정보보호관리체계)

  • ISMS-P: 정보보호 + 개인정보보호 인증
  • 관리체계 수립 → 보호대책 구현 → 지속 개선
  • 법적 의무 대상: 매출 100억+ ISP, 병원 등
ISO 27001
국제 정보보안 관리체계 표준
Plan-Do-Check-Act (PDCA) 기반

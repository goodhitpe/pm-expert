Project Quality Management

📅 교육 6일차⏱️ 8시간✅ QA · QC · 7가지 품질 도구

📌 품질 vs 등급

✅ 품질 (Quality)

요구사항을 충족하는 정도

낮은 품질 = 항상 문제

예: 버그 많은 소프트웨어

📊 등급 (Grade)

기능적 용도의 범주

낮은 등급 ≠ 문제

예: 이코노미 vs 비즈니스 클래스

품질 관리의 3대 프로세스

1. 품질 관리 계획 수립 → 2. 품질 관리(QA): 프로세스 올바른가? → 3. 품질 통제(QC): 산출물 기준 충족?

💰 품질 비용 (COQ)

👍 적합 비용

🛡️ 예방 비용 (Prevention)
교육, 프로세스 문서화, 장비 보정, 설계 검토
🔍 평가 비용 (Appraisal)
테스팅, 검사, 감사, 코드 리뷰

👎 부적합 비용

⚠️ 내부 실패 비용
재작업, 폐기, 결함 수정 (출시 전 발견)
🔥 외부 실패 비용
보증, 소송, 리콜, 신뢰 상실 (출시 후 발견)
핵심 원칙

예방 비용 투자가 전체 COQ를 줄입니다. 결함은 일찍 발견할수록 수정 비용이 낮습니다! (1:10:100 법칙)

📊 QA vs QC

🔍 QA (품질 보증)

  • 프로세스에 초점
  • 예방적 (Proactive)
  • "올바른 방법으로 하고 있는가?"
  • 품질 감사(Audit) 수행
  • 지속적 프로세스 개선

✅ QC (품질 통제)

  • 산출물(제품)에 초점
  • 탐지적 (Reactive)
  • "결과물이 기준을 충족하는가?"
  • 검사, 테스팅 수행
  • 결함 발견 및 수정

파레토 원칙 (80/20 Rule)

20%의 원인이 80%의 결함을 유발합니다. 빈도순 막대그래프 + 누적 % 곡선으로 핵심 원인을 파악합니다.

관리도 - Rule of Seven

7개 연속 데이터가 평균의 한쪽에 위치하면 → 프로세스가 비정상 (조사 필요!)

🔄 PDCA & 6시그마

PDCA 사이클 (데밍 사이클)

📝 Plan (계획) → ⚡ Do (실행) → 🔍 Check (확인) → 📋 Act (조치)

지속적 개선의 기본 프레임워크. 반복 수행합니다.

6시그마 DMAIC

Define → Measure → Analyze → Improve → Control

100만개 중 3.4개 결함 목표. 데이터 기반 프로세스 개선.

CMMI 5단계

초기 → 관리 → 정의 → 정량적 관리 → 최적화

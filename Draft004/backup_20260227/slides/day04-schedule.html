🔗 선후행 관계 (PDM)

Precedence Diagramming Method - 활동 간 의존 관계 유형

유형 설명 빈도 예시
FS 선행 완료 → 후행 시작 가장 일반적 기초 완공 → 골조 시작
FF 선행 완료 → 후행 완료 보통 코딩 완료 → 문서화 완료
SS 선행 시작 → 후행 시작 보통 설계 시작 → 검토 시작
SF 선행 시작 → 후행 완료 매우 드묾 신 시스템 시작 → 구 시스템 종료
리드(Lead) vs 래그(Lag)

리드: 후행 활동을 앞당기는 시간 (FS -2일) | 래그: 후행 활동을 지연시키는 대기 시간 (FS +3일)

🔴 CPM (Critical Path Method)

주공정(Critical Path)이란?

프로젝트를 완료하기까지 가장 긴 경로. 이 경로의 활동이 지연되면 전체 프로젝트가 지연됩니다.

핵심 용어

  • ES (Early Start): 가장 빠른 시작일
  • EF (Early Finish): 가장 빠른 종료일
  • LS (Late Start): 가장 늦은 시작일
  • LF (Late Finish): 가장 늦은 종료일
  • TF (Total Float): 총 여유 = LS - ES
  • FF (Free Float): 자유 여유

계산 방법

전진 계산 (Forward Pass)
ES, EF 계산 → 프로젝트 최소 기간
후진 계산 (Backward Pass)
LS, LF 계산 → 여유 시간 산출
Total Float = LS - ES = LF - EF
TF = 0 → 주공정 활동!

⚡ 일정 단축 기법

💰 공정 압축 (Crashing)

  • 자원 추가 투입으로 기간 단축
  • 비용 증가 수반
  • 주공정 활동에만 적용 의미 있음
  • 비용 대비 효과 분석 필요
비용 기울기 = (단축비용 - 정상비용) / (정상기간 - 단축기간)

🏃 빠른 추적 (Fast-tracking)

  • 순차적 활동을 병행 수행
  • 재작업 리스크 증가
  • FS 관계를 SS 관계로 전환
  • 비용은 증가하지 않을 수 있음

병행 수행으로 인한 오류 발생 시 재작업 비용이 더 클 수 있음!

❓ PERT 계산 연습

낙관=4일, 최빈=6일, 비관=14일일 때:
베타분포: (4 + 4×6 + 14) / 6 = 7일
표준편차: (14 - 4) / 6 = 1.67일

🗄️ 데이터베이스란?

정의

체계적으로 조직·관리되는 데이터의 집합과 이를 관리하는 시스템 (DBMS)

RDBMS 핵심 용어

용어 설명 비유
테이블 (Table) 데이터를 저장하는 2차원 구조 엑셀 시트
행 (Row / Record) 하나의 데이터 레코드 엑셀의 한 행
열 (Column / Field) 데이터의 속성 엑셀의 열 헤더
기본키 (PK) 각 행을 유일하게 식별하는 키 주민번호, 사원번호
외래키 (FK) 다른 테이블의 PK를 참조 테이블 간 연결고리
인덱스 (Index) 검색 속도를 높이는 데이터 구조 책의 색인(찾아보기)

📐 ERD (Entity-Relationship Diagram)

ERD란?

DB의 엔티티(테이블)간 관계를 시각적으로 표현한 다이어그램. 데이터 모델링의 핵심 도구

관계 유형

1️⃣↔1️⃣
1:1 (일대일)
사원 ↔ 사원상세
한 사원은 하나의 상세정보만
1️⃣↔♾️
1:N (일대다)
부서 ↔ 사원
한 부서에 여러 사원
가장 일반적
♾️↔♾️
N:M (다대다)
학생 ↔ 과목
중간 테이블로 분리
(수강 테이블)
PM 관점에서 ERD가 중요한 이유

데이터 구조를 이해 → 마이그레이션 범위, 연동 복잡도, 성능 이슈를 사전에 파악 가능!

📝 SQL 기초

CRUD (Create, Read, Update, Delete)

작업 SQL 문 예문
조회 (Read) SELECT SELECT 이름, 부서 FROM 사원 WHERE 직급 = '과장'
삽입 (Create) INSERT INSERT INTO 사원 (이름, 부서) VALUES ('홍길동', '개발팀')
수정 (Update) UPDATE UPDATE 사원 SET 부서 = '기획팀' WHERE 사원번호 = 101
삭제 (Delete) DELETE DELETE FROM 사원 WHERE 퇴사일 IS NOT NULL
JOIN (테이블 결합)

INNER JOIN: 양쪽 모두 일치하는 데이터만 | LEFT JOIN: 왼쪽 테이블 전체 + 오른쪽 일치

📊 정규화 (Normalization)

정규화란?

데이터 중복을 최소화하고 이상(Anomaly)을 방지하기 위해 테이블을 분리하는 과정

정규형 조건 핵심
1NF 원자값, 반복 그룹 제거 한 칸에 하나의 값만!
2NF 1NF + 부분 함수 종속 제거 PK 전체에 종속
3NF 2NF + 이행 함수 종속 제거 PK에만 직접 종속
BCNF 모든 결정자가 후보키 더 강한 3NF
이상(Anomaly)

삽입 이상: 불필요한 데이터 함께 삽입
갱신 이상: 일부만 수정 → 불일치
삭제 이상: 필요 데이터 함께 삭제

반정규화 (Denormalization)

성능 향상을 위해 의도적으로 중복 허용. 조회 속도 ↑, 데이터 일관성 관리 복잡 ↑

🔀 NoSQL 데이터베이스

Not Only SQL — 관계형이 아닌 다양한 데이터 모델

📋 문서형 (Document)
JSON/BSON 문서 저장
유연한 스키마
MongoDB, CouchDB
🔑 키-값 (Key-Value)
Key로 Value 조회
초고속 읽기/쓰기
Redis, DynamoDB
📊 컬럼형 (Column-family)
컬럼 기반 저장
대규모 분석에 적합
Cassandra, HBase
🕸️ 그래프 (Graph)
노드·엣지 관계 저장
소셜 네트워크, 추천
Neo4j, Amazon Neptune
RDBMS vs NoSQL 선택 기준

RDBMS: 정형 데이터, 트랜잭션 중요, 복잡한 JOIN → ERP, 금융
NoSQL: 비정형 데이터, 대용량, 유연한 스키마, 수평 확장 → 소셜, IoT, 로그

🏗️ 데이터 아키텍처

트랜잭션 & ACID

ACID 속성
Atomicity (원자성): 전부 또는 전무
Consistency (일관성): 규칙 유지
Isolation (격리성): 동시 간섭 차단
Durability (지속성): 영구 저장

금융 거래 등 데이터 정합성이 중요한 경우 ACID 보장이 필수!

데이터 웨어하우스 & 레이크

🏢 데이터 웨어하우스
정형 데이터를 통합·정제하여 분석용으로 저장. ETL 프로세스 활용
🌊 데이터 레이크
원시 데이터를 형태 그대로 저장. 정형+비정형 모두 수용. 빅데이터 분석

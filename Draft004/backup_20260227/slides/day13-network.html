Network Fundamentals for PM

🌐 OSI 7계층 모델

계층 이름 핵심 기능 프로토콜/장비 PM 관점
7 응용(Application) 사용자 인터페이스 HTTP, SMTP, FTP API, 웹 서비스
6 표현(Presentation) 데이터 변환/암호화 SSL/TLS, JPEG 보안 인증서
5 세션(Session) 연결 관리 RPC, NetBIOS 세션 타임아웃
4 전송(Transport) 신뢰성 있는 전송 TCP, UDP 포트 관리
3 네트워크(Network) 라우팅, IP 주소 IP, ICMP, Router 서브넷, 방화벽
2 데이터링크 프레임 전송 Ethernet, Switch VLAN 구성
1 물리(Physical) 전기 신호 전송 케이블, Hub 인프라 구성

PM이 주로 다루는 영역: L7(애플리케이션), L4(성능), L3(방화벽/보안) 수준의 이해가 중요

📡 TCP vs UDP

🔒 TCP (Transmission Control Protocol)

  • 연결형 (3-Way Handshake)
  • 신뢰성 보장 (순서, 재전송)
  • 속도는 상대적으로 느림
  • 흐름 제어, 혼잡 제어

용도: 웹(HTTP), 이메일, 파일 전송, DB 통신

⚡ UDP (User Datagram Protocol)

  • 비연결형 (연결 설정 없음)
  • 속도 우선 (순서 보장 X)
  • 패킷 손실 가능
  • 실시간 전송에 적합

용도: 동영상 스트리밍, 게임, VoIP, DNS 조회

🌍 DNS & HTTP

DNS (Domain Name System)

www.example.com → DNS 서버 → 93.184.216.34
  • 도메인 이름을 IP 주소로 변환
  • 계층 구조: Root → TLD(.com) → Domain
  • PM 관점: DNS 전파 시간, 도메인 이전

HTTP/HTTPS

HTTP 메서드
GET 조회 | POST 생성 | PUT 수정 | DELETE 삭제
상태 코드
200 OK | 301 Redirect | 404 Not Found | 500 Server Error

HTTPS = HTTP + SSL/TLS 암호화
개인정보 취급 시 필수!

🔌 REST API

API (Application Programming Interface)

서로 다른 시스템이 데이터를 주고받는 규약. 현대 IT 시스템의 핵심 연결고리

REST API 핵심 원칙

🌐 자원 기반 (Resource-based)
URL로 자원 식별
/api/users/123
📋 HTTP 메서드 활용
CRUD와 매핑
GET·POST·PUT·DELETE
🔄 무상태 (Stateless)
각 요청은 독립적
서버에 상태 저장 X
📦 JSON 응답
표준 데이터 형식으로
시스템 간 연동 용이
PM 관점에서 API가 중요한 이유

시스템 연동 범위, API 설계 일정, 외부 API 의존성, API 성능 SLA 등이 프로젝트 범위·일정·리스크에 직접 영향!

☁️ 클라우드 인프라

IaaS
Infrastructure as a Service
서버, 네트워크, 스토리지
AWS EC2, Azure VM
PaaS
Platform as a Service
개발·배포 플랫폼
Heroku, Google App Engine
SaaS
Software as a Service
완전한 소프트웨어 서비스
Gmail, Salesforce, Slack
PM이 클라우드를 알아야 하는 이유

• 인프라 조달 일정 단축 (몇 주 → 몇 분)
• 원가 모델 변화 (CAPEX → OPEX)
• SLA/가용성 요구사항 정의
• 보안·컴플라이언스 고려사항

📊 네트워크 성능 지표 & SLA

지표 설명 일반적 기준
가용성(Availability) 시스템 정상 운영 시간 비율 99.9% (8.7시간/년 다운)
응답시간(Latency) 요청~응답까지 소요 시간 웹: 2초 이내
처리량(Throughput) 단위 시간당 처리 건수 TPS 기준 정의
동시 사용자 동시 접속 지원 수 프로젝트별 정의
가용성 99.99% (Four Nines) = 연간 52.6분 다운타임 허용

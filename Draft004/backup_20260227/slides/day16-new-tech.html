• 반복적·탐색적 개발 → 애자일 적합

🔗 블록체인

핵심 개념

분산 원장 기술 — 거래 기록을 모든 참여자가 공유하며, 변조가 사실상 불가능한 구조

🔐 불변성
한번 기록된 데이터는 수정·삭제 불가
🌐 분산화
중앙 기관 없이 P2P 네트워크로 운영
📋 스마트 계약
조건 충족 시 자동 실행되는 프로그래밍 가능한 계약
🏢 활용 분야
금융, 공급망 추적, 디지털 자산, 전자 투표, 인증
PM 관점의 블록체인 프로젝트 주의점

• 모든 문제에 블록체인이 필요한 것은 아님! (적합성 평가 필수)
• 성능 제약 (TPS 한계), 에너지 소비, 규제 불확실성 고려

📡 IoT (Internet of Things)

정의

물리적 장치에 센서·네트워크를 부착하여 데이터를 수집·교환하는 기술 생태계

IoT 아키텍처

Layer 1
🔌 디바이스 (센서, 액추에이터)
Layer 2
🌐 네트워크 (Wi-Fi, 5G, LoRa)
Layer 3
☁️ 플랫폼 (데이터 처리·분석)
Layer 4
📱 애플리케이션 (대시보드, 알림)

활용 분야

🏭 스마트 팩토리: 설비 모니터링, 예지 정비
🏠 스마트 홈: 에너지 관리, 보안
🚗 커넥티드 카: 차량 원격 진단
🏥 헬스케어: 원격 환자 모니터링

개발(Dev)과 운영(Ops)의 벽을 허물어, 소프트웨어 인도 속도와 품질을 높이는 문화·방법론

🔄 CI (Continuous Integration)
코드 변경 시 자동으로 빌드·테스트
통합 오류를 조기 발견
Jenkins, GitHub Actions, GitLab CI
🚀 CD (Continuous Delivery/Deployment)
자동으로 스테이징·프로덕션 배포
릴리스 주기 대폭 단축

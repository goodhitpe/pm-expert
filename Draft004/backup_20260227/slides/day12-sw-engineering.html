🏃 애자일(Agile) 핵심

애자일 선언문 4가지 가치

프로세스보다 개인과 상호작용 | 문서보다 동작하는 소프트웨어 | 계약 협상보다 고객 협력 | 계획 따르기보다 변화 대응

스크럼(Scrum) 프레임워크

역할
Product Owner · Scrum Master · Development Team
이벤트
Sprint Planning · Daily Standup · Sprint Review · Retrospective
산출물
Product Backlog · Sprint Backlog · Increment

칸반(Kanban)

핵심 원칙
• 작업을 시각화 (보드)
• WIP 제한 (동시 작업 수 제한)
• 흐름 관리 (리드타임 최적화)
• 지속적 개선

스크럼: 고정 Sprint → 변화에 빠르게 대응
칸반: 연속 흐름 → 운영·유지보수에 적합

🔍 소프트웨어 테스팅

테스트 레벨 범위 수행자 목적
단위 테스트 함수/클래스 개발자 개별 모듈 정상 동작 확인
통합 테스트 모듈 간 연동 개발/QA 모듈 인터페이스 검증
시스템 테스트 전체 시스템 QA팀 요구사항 충족 여부 검증
인수 테스트 (UAT) 비즈니스 검증 사용자/고객 비즈니스 요구 충족 확인
블랙박스 테스트

내부 구조 모르는 상태에서 입·출력만으로 테스트 (경계값, 동치분할)

화이트박스 테스트

내부 코드 구조를 보고 경로·조건을 테스트 (코드 커버리지)

📋 요구사항 & 형상관리

요구사항 분류

📋 기능 요구사항 (Functional)
"시스템이 무엇을 해야 하는가?"
로그인, 검색, 주문 기능 등
⚡ 비기능 요구사항 (Non-functional)
"시스템이 어떻게 동작해야 하는가?"
성능, 보안, 확장성, 가용성 등

PM은 비기능 요구사항을 놓치지 마세요!
성능 SLA, 보안 기준, 동시 사용자 수 등

형상관리 (Configuration Management)

  • 식별: 형상 항목 정의
  • 통제: 변경 요청 → 승인 → 반영
  • 기록: 변경 이력 추적
  • 감사: 형상 항목 무결성 검증
Git을 활용한 형상관리

Branch 전략, Pull Request, Code Review가 현대 SW 개발의 핵심입니다.

🔐 DevSecOps & SAFe

🛡️ DevSecOps (Shift-Left)

보안을 개발 마지막이 아닌 처음부터 내재화

  • SAST: 소스코드 정적 분석
  • DAST: 실행 중 동적 분석
  • SCA: 오픈소스 취약점 점검
  • CI/CD 파이프라인에 보안 게이트 삽입

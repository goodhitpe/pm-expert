P-I 매트릭스

🔴 즉시 대응 🟡 감시 🟢 수용 가능

리스크 점수 계산 예시

리스크 점수 = 확률 × 영향
리스크확률영향점수우선순위
핵심 개발자 이탈 중간 (0.5) 높음 (0.8) 0.40 🔴 1위
요구사항 불명확 높음 (0.7) 중간 (0.5) 0.35 🟡 2위
3rd-party API 장애 낮음 (0.2) 높음 (0.8) 0.16 🟡 3위
서버 비용 초과 낮음 (0.2) 중간 (0.4) 0.08 🟢 4위
리스크 긴급도(Urgency)
점수가 같아도 근접성(Proximity)이 높으면 우선 대응
예: 2주 내 발생 가능한 리스크 = 6개월 후보다 긴급

⏱ 1.5시간

정량적 리스크 분석 기법

EMV & 의사결정 나무

EMV = 확률 × 금액(화폐가치)
의사결정 나무 예시
□ 자체 개발 vs 구매 결정
○ 성공 확률 70% → 수익 +5억
○ 실패 확률 30% → 손실 -3억
EMV = 0.7×5억 + 0.3×(-3억) = 3.5억 - 0.9억 = 2.6억
EMV 적용 예:
리스크 확률 30%, 손실 1억원
EMV = 0.3 × -1억 = -3,000만원
→ 보험료가 3,000만원 미만이면 가입 합리적

토네이도 & 몬테카를로

토네이도 다이어그램 (Tornado Diagram)
민감도 분석 시각화
막대가 길수록 → 프로젝트 결과에 더 큰 영향
맨 위 = 가장 중요한 리스크 변수

몬테카를로 시뮬레이션
각 활동 기간을 확률 분포로 정의
→ 수천 번 반복 시뮬레이션
→ 전체 일정/원가의 확률 분포(S-Curve)
예: "80% 확률로 150일 내 완료"
고위험/대규모 프로젝트에 적용

학습 확인 퀴즈

Q1. 리스크와 이슈(Issue)의 차이를 정의하고, 같은 사건이 어떻게 리스크에서 이슈로 전환되는지 예를 들어 설명하시오.

▶ 모범 답안

리스크: 아직 발생하지 않은 불확실한 사건 → 리스크 등록부(Risk Register)에 기록
이슈: 이미 발생한 문제 → 이슈 로그(Issue Log)에 기록, 즉각 대응

전환 예: "3rd-party API가 업데이트로 호환성이 깨질 수 있다" (리스크)
→ "어제 3rd-party API가 업데이트되어 우리 서비스가 다운됐다" (이슈)

Q2. 기회(Opportunity)에 대한 4가지 대응 전략을 각각 한 단어와 예시로 설명하시오.

▶ 모범 답안

활용(Exploit): 기회 실현 보장 — 최고 전문가 배정으로 조기 완료 확정
공유(Share): 제3자와 협력 — JV 파트너십으로 시장 선점 기회 공동 추구
증대(Enhance): 확률·영향 증대 — 추가 자원 투입으로 신기술 우선 도입 가능성 높임
수용(Accept): 인정하되 적극 추구 않음 — 여건이 되면 활용 기록

Q3. 리스크 EMV가 -2,000만원이다. 보험료 500만원의 보험에 가입하는 것이 합리적인지 분석하시오.

▶ 모범 답안

EMV -2,000만원 = 해당 리스크 발생 시 기대 손실이 2,000만원
보험료 500만원 < 기대 손실 2,000만원
→ 보험 가입이 합리적 (비용 < 기대 손실 제거 효과)

추가 고려: 보험 면제 조항, 잔여 리스크, 2차 리스크(보험사 파산) 검토 필요
이는 리스크 전가(Transfer) 전략에 해당합니다.

Q4. 잔여 리스크와 2차 리스크의 차이를 설명하고, 각각에 대한 PM의 대응 방법을 설명하시오.

▶ 모범 답안

잔여 리스크: 대응 조치 후에도 남아있는 리스크
대응: 우발 사태 예비비(Contingency Reserve)로 대비. 사전에 수량화하여 예산에 반영

2차 리스크: 대응 조치로 인해 새로 발생한 리스크
대응: 리스크 등록부에 새로운 리스크로 추가 등록. 동일한 분석·대응 프로세스 적용

예: 아웃소싱(Transfer) → 벤더 파산 리스크(2차) 발생 → 새로운 리스크로 관리

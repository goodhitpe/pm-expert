이해관계자 참여 계획 수립

참여도 매트릭스 · C(현재) vs D(희망) · 전략 수립

⏱ 1.5시간

이해관계자 참여도 매트릭스

5단계 참여 수준

불인지 (Unaware): 프로젝트 존재 자체를 모름
저항 (Resistant): 변화에 반대, 방해 가능성
중립 (Neutral): 지지도 반대도 아님
지지 (Supportive): 프로젝트를 돕겠다는 의지
주도 (Leading): 적극 옹호, Champion 역할

참여도 매트릭스 (C:현재 / D:목표)

이해관계자불인지저항중립지지주도
스폰서 C D
현업부서장 C D
최종사용자 C D
IT팀장 C D
💡 C = D → 이상적, 현재 참여도가 목표와 일치
C ≠ D → 갭 분석 후 전략 수립 필요
저항: 원인 파악 → 우려 해소 → 점진적 참여 유도
불인지: 존재 알리기 → 이해 증진 → 이익 공유

🏆 Champion (프로젝트 옹호자)

프로젝트를 적극 지지하고 조직 내 홍보·설득
PM이 접근 어려운 경영진에 대한 영향력 행사
예: C-Level 스폰서가 Champion이 되면
예산·우선순위 확보가 훨씬 쉬워짐
기대 관리 3원칙
1. 현실적 기대 설정: 과도한 약속 금지
2. 투명한 소통: 나쁜 소식도 빠르게
3. 일관성 유지: 말과 행동이 일치해야 신뢰 형성
⚠️ 이해관계자가 원하는 것(Wants)과
실제 필요한 것(Needs)은 다를 수 있음
진짜 문제를 파악하는 능동적 경청 필수

학습 확인 퀴즈

Q1. 권력-관심도 매트릭스의 4개 사분면별 관리 전략을 설명하고, 각 사분면에 해당하는 이해관계자 예시를 드시오.

▶ 모범 답안

고권력+고관심 → Manage Closely (긴밀 관리): 스폰서, 핵심 고객 — 정기 미팅, 모든 주요 결정 참여
고권력+저관심 → Keep Satisfied (만족 유지): CEO, 경영진 — 주요 이슈만 보고, 과도한 세부 사항 피함
저권력+고관심 → Keep Informed (정보 제공): 최종 사용자, 사용자 부서 — 진행 현황 공유
저권력+저관심 → Monitor (모니터링): 외부 감사, 규제 기관 — 변화 발생 시만 알림

Q2. 저항하는 이해관계자를 지지 수준으로 전환하기 위한 전략 3가지를 구체적으로 제시하시오.

▶ 모범 답안

① 저항 원인 파악: 개인 면담을 통해 실제 우려 사항 파악. 변화에 따른 역할 변화, 업무 부담 증가 등 구체적 이유 도출
② 우려 사항 해소 및 참여 유도: 공식적으로 우려 기록 + 대응 계획 공유. 파일럿 참여 제안 — 작은 성공 체험 제공
③ 의사결정 참여 보장: 주요 결정에 해당 이해관계자를 포함. "당신의 의견이 반영되었다"는 인식을 줌 → 소유의식(Ownership) 형성

Q3. 이해관계자 참여도 매트릭스에서 C(현재)와 D(목표)가 다를 때 PM이 해야 할 일을 설명하시오.

▶ 모범 답안

C ≠ D는 현재 참여도가 프로젝트 성공을 위한 목표 수준에 미달함을 의미합니다.
① 갭 분석: 어떤 이해관계자의 어떤 단계에서 갭이 있는지 구체화
② 원인 파악: 정보 부족인지, 변화 저항인지, 관계 문제인지 진단
③ 맞춤 전략 수립: 갭 유형에 따라 소통 방식·빈도·내용 조정
④ 진행 추적: 전략 실행 후 참여도 변화 재평가 (주기적 등록부 갱신)

Q4. 이해관계자 식별이 Day 07 RACI 차트, Day 08 의사소통 계획과 어떻게 연결되는지 설명하시오.

▶ 모범 답안

이해관계자 식별 → RACI 차트:
이해관계자 등록부에서 각 이해관계자의 역할과 권한 수준이 파악되면
RACI의 R/A/C/I 배정의 기초가 됩니다. 특히 Accountable(A)은 고권력 이해관계자에게 배정

이해관계자 식별 → 의사소통 계획:
이해관계자 등록부의 P-I 위치와 정보 요구사항이
의사소통 매트릭스의 '이해관계자' 열의 直접적 입력이 됩니다.
Keep Informed → 정기 보고(Push) 방식, Manage Closely → Interactive 방식 선택

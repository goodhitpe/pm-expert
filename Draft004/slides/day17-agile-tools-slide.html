3가지 역할 (Roles)

제품 비전 수립 · 제품 백로그 우선순위 결정 · ROI 책임

학습 확인 퀴즈

Q1. 스크럼의 3가지 역할·5가지 이벤트·3가지 산출물을 모두 나열하고, PO와 SM의 역할 차이를 설명하시오.

▶ 모범 답안

역할 3가지: Product Owner (PO), Scrum Master (SM), 개발팀
이벤트 5가지: Sprint, Sprint Planning, Daily Scrum, Sprint Review, Sprint Retrospective
산출물 3가지: 제품 백로그, 스프린트 백로그, 증분(Increment)

PO vs SM 차이:
PO: "무엇을 만들 것인가" — 제품 비전·백로그 우선순위·ROI 책임. 고객·사용자·비즈니스 대변자
SM: "어떻게 만들 것인가" — 프로세스 코치, 장애물 제거, 팀 보호. 애자일 가치를 조직에 적용하는 촉진자 (관리자 역할 ≠ SM)

Q2. 칸반의 WIP 제한이 왜 필요한지 설명하고, 워터폴·스크럼·칸반 중 각각 적합한 프로젝트 사례를 하나씩 들어 설명하시오.

▶ 모범 답안

WIP 제한 필요 이유: 동시 작업이 많아지면 컨텍스트 전환(Context Switching) 비용 증가, 특정 단계에서 병목 발생. WIP를 제한하면 팀이 현재 작업에 집중하고, 병목이 가시화되어 즉각 대응 가능. 결과적으로 리드타임 단축

워터폴: 건설·제조·국방 시스템 — 요구사항이 법·규정으로 확정되고, 중간 변경이 막대한 비용을 유발하는 프로젝트
스크럼: 신규 앱·서비스 개발 — 사용자 요구가 불확실하고, 빠른 가치 인도와 피드백 반영이 핵심인 프로젝트
칸반: IT 운영팀·헬프데스크·유지보수 — 작업이 불규칙적으로 유입되고 스프린트 리듬보다 지속적 처리가 중요한 환경

Q3. EVM 지표 중 CPI = 0.8의 의미를 설명하고, Jira와 MS Project 중 이를 추적하기에 적합한 도구와 이유를 설명하시오.

▶ 모범 답안

CPI = 0.8의 의미: 비용 성과 지수(Cost Performance Index) = EV/AC = 0.8
1달러(원) 예산에 0.8달러어치 가치만 생산 → 20% 비용 초과 상태
1보다 작을수록 예산 낭비. 조기 경고 시그널로 EAC = BAC/CPI 공식으로 완료 시 예측 비용 계산 가능

적합 도구: MS Project
이유: MS Project는 기준선(Baseline) 저장 후 실제 진척 입력 시 EVM 지표(CPI·SPI·EAC·ETC)를 자동 계산하고 보고서를 생성함. Jira는 스토리 포인트 기반 번다운 차트에 특화되어 있어 비용 기반 EVM 추적에는 MS Project가 더 적합함

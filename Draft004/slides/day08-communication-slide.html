의사소통 방법 3가지

Interactive (상호작용형)
회의, 전화, 화상회의, 즉각 응답
양방향 실시간 — 복잡한 이슈 해결에 최적
Push (밀어넣기형)
이메일, 보고서 배포, 공지사항, 메모
발신자 → 수신자 (수신 확인 없음)
Pull (당기기형)
공유 드라이브, 인트라넷, 지식 저장소
수신자가 필요 시 접근 — 대규모 정보 배포
💡 선택 기준: 긴급성/민감도/분산도/청중 수
긴급 + 복잡 → Interactive
정기 배포 + 기록 필요 → Push
대용량 + 참조용 → Pull

⏱ 1.5시간

효과적 의사소통 실행

메시지 전달 — 55-38-7 법칙

능동적 경청 (Active Listening)
• 눈 맞춤, 고개 끄덕임, 집중 자세
• 확인 질문: "말씀하신 게 ~이 맞나요?"
• 요약 반복: "핵심을 정리하면 ~입니다"
• 판단 유보 — 끝까지 듣기

효과적 회의 운영 3단계

회의 전 (Before)
• 안건(Agenda) 48시간 전 배포
• 목적·기대 결과 명확화
• 필요 인원만 초대 (채널 수 최소화!)
회의 중 (During)
• 타임키핑 + 안건 준수
• 참여 유도, 침묵 방지
• 회의록(Minutes) 실시간 작성
회의 후 (After)
• 24시간 내 회의록 배포
• Action Item: 담당자·기한 명시
• 다음 회의 전 ACTION 진행 확인
⚠️ 불필요한 회의 = 생산성 최대 적
회의 필요성 체크: 이메일/문서로 대체 가능한가?

학습 확인 퀴즈

Q1. 프로젝트 팀원이 6명에서 10명으로 증가했을 때 의사소통 채널 수는 몇 개 증가하는가?

▶ 모범 답안

6명: 6×5÷2 = 15채널
10명: 10×9÷2 = 45채널
증가량: 45 - 15 = 30채널 증가
4명 추가로 채널이 2배 증가 → 의사소통 복잡도 관리 필요성 강조

Q2. Push, Pull, Interactive 방법의 차이를 각각 예시 1개와 함께 설명하시오.

▶ 모범 답안

Interactive: 양방향 실시간 — 예: 화상회의, 긴급 이슈 전화
수신 확인이 가장 강력. 복잡한 사안 해결에 적합

Push: 발신자 배포, 수신 선택은 수신자 — 예: 주간 보고서 이메일
기록이 남으나 읽었는지 보장 없음

Pull: 수신자가 필요 시 접근 — 예: SharePoint 프로젝트 문서 저장소
대용량·다수 배포에 효율적

Q3. 작업 성과 데이터, 작업 성과 정보, 작업 성과 보고서의 차이를 설명하시오.

▶ 모범 답안

데이터: 가공 전 원시 수치 — 예: "완료 작업 30개, 지출 3억"
정보: 분석 후 의미 있는 지표 — 예: "SPI=0.85 (일정 지연), CV=-0.5억"
보고서: 의사결정을 위한 종합 문서 — 예: "현황판단 + 원인분석 + 권고안"
순서: 데이터(실행) → 정보(모니터링) → 보고서(의사결정)

Q4. 메라비안의 55-38-7 법칙이 PM 의사소통에서 갖는 의미는?

▶ 모범 답안

말하는 내용(단어)은 전체 인상의 7%만 차지하며, 목소리 톤·속도가 38%, 비언어(표정·자세·제스처)가 55%를 차지합니다.
PM 실무 시사점: 이메일/채팅(단어만 전달)은 오해 위험이 높음 → 중요도가 높거나 민감한 사안은 반드시 대면/화상 미팅으로 진행해야 합니다.

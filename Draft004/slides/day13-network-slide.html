DNS · HTTP · REST API

DNS (Domain Name System)
도메인 이름 → IP 주소 변환 "인터넷 전화번호부"

example.com → 93.184.216.34

DNS 계층: 루트(.) → TLD(.com/.kr) → SLD(example) → 호스트(www)

주요 레코드:
A: 도메인→IPv4 · AAAA: 도메인→IPv6
CNAME: 별칭 · MX: 메일서버
TXT: 텍스트(SPF, 도메인 인증)
HTTP 상태 코드 분류
2xx = 성공 (200 OK, 201 Created)
3xx = 리다이렉트 (301 영구이동)
4xx = 클라이언트 오류 (404 Not Found)
5xx = 서버 오류 (500 Internal Server Error)

학습 확인 퀴즈

Q1. OSI 7계층을 1번(물리)부터 7번(응용)까지 순서대로 나열하고, 4번(전송) 계층과 3번(네트워크) 계층의 핵심 차이를 설명하시오.

▶ 모범 답안

1-물리 / 2-데이터 링크 / 3-네트워크 / 4-전송 / 5-세션 / 6-표현 / 7-응용
(암기: 물데네전세표응)

3계층(네트워크): IP 주소 기반 라우팅 — "어느 컴퓨터로 보낼지" (경로 결정), 라우터 사용
4계층(전송): TCP/UDP, 포트 번호 기반 — "어느 프로그램으로 보낼지" (신뢰성·순서 보장), 소켓 통신

Q2. TCP와 UDP를 비교하고, 각각이 적합한 서비스 유형을 2가지씩 예시와 함께 설명하시오.

▶ 모범 답안

TCP: 연결 지향, 3-Way Handshake, 순서 보장, 오류 재전송 — 신뢰성 높지만 느림
적합 서비스: ①은행 거래 시스템 (데이터 누락 절대 불가) ②이메일(SMTP) (순서·완전성 중요)

UDP: 비연결, Handshake 없음, 빠르지만 신뢰성 없음
적합 서비스: ①화상회의/실시간 스트리밍 (약간의 패킷 손실보다 지연이 더 나쁨) ②DNS (빠른 응답 우선)

Q3. IaaS, PaaS, SaaS의 차이를 설명하고, IT 프로젝트에서 PM이 각 모델 선택 시 고려해야 할 사항을 기술하시오.

▶ 모범 답안

IaaS: 가상 서버/스토리지만 제공 (AWS EC2, Azure VM) — OS·미들웨어·SW 직접 관리. PM 고려: 인프라 관리 팀 역량, 운영 인력 비용 포함

PaaS: 개발 플랫폼 제공 (Heroku) — 애플리케이션 개발만 집중. PM 고려: 특정 플랫폼 벤더 종속(Lock-in) 리스크, 확장성 한계

SaaS: 완성 SW 사용 (Slack, Gmail) — 별도 개발 불필요. PM 고려: 커스터마이징 제한, 데이터 이관 계획, 계약·SLA 검토

공통 고려 사항: CAPEX(온프레미스) vs OPEX(클라우드) 비용 구조, 데이터 보안·규정 준수, 가용성 SLA

Q4. IDS와 IPS의 차이를 설명하고, PM이 이를 보안 요구사항 정의 시 어떻게 활용하는지 서술하시오.

▶ 모범 답안

IDS (침입 탐지 시스템): 이상 패턴을 탐지하여 경보만 발송. 직접 차단 불가 — 사람이 확인 후 조치 필요

IPS (침입 방지 시스템): 이상 패턴 탐지 + 자동 차단. 실시간 방어 가능 — 오탐(False Positive) 시 정상 트래픽 차단 위험

PM 활용법:
① 보안 요구사항 단계에서 "IDS/IPS 구축 요구"를 NFR로 명시 (Day12 연계)
② RFP에 IDS/IPS 요구사항 포함 (Day10 조달 연계)
③ 도입 후 경보 응답 시간 = SLA 지표로 관리
④ IPS 오탐으로 인한 서비스 중단 = 리스크 등록부에 포함 (Day09 연계)

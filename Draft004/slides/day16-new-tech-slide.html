컨테이너 & 서버리스 — 현대 클라우드 핵심

🐳 컨테이너 (Docker + Kubernetes)

  • Docker: 앱을 격리된 컨테이너로 패키징 — "내 PC에선 됩니다" 문제 해결
  • Kubernetes(K8s): 컨테이너 오케스트레이션 — 배포·스케일링·복구 자동화
  • MSA 연계: 마이크로서비스 각각을 개별 컨테이너로 운영
  • 장점: 이식성 ↑, 환경 일관성, 빠른 배포
  • PM 관점: DevOps 파이프라인과 통합 필수

⚡ 서버리스 (Serverless)

  • FaaS (Function as a Service): AWS Lambda, Azure Functions
  • 서버 관리 불필요 — 함수 단위 실행
  • 이벤트 기반 아키텍처 (API 호출·파일 업로드·스케줄)
  • 장점: 운영 부담 최소화, 사용량 기반 과금
  • 단점: 콜드 스타트 지연, 복잡한 시스템에 부적합
클라우드 3대 플랫폼: AWS EC2·S3·Lambda·RDS Azure VM·Blob·Functions·SQL DB GCP Compute Engine·BigQuery·Cloud Run

클라우드 마이그레이션 전략 — 6R

전략별칭방법PM 적용 시기
Rehost Lift & Shift 코드 변경 없이 클라우드로 이전 빠른 마이그레이션 우선, 최소 리스크
Replatform Lift, Tinker & Shift 소규모 최적화 후 이전 (예: DB → RDS) 성능 개선이 필요하나 대규모 변경은 곤란할 때
Repurchase Drop & Shop SaaS 솔루션으로 교체 (예: CRM → Salesforce) 레거시를 SaaS로 대체 — 라이선스 전환
Refactor Re-architect 클라우드 네이티브로 전면 재설계 (MSA) 장기적 확장성 목표, 투자 비용 큼
Retire - 사용하지 않는 시스템 폐기 레거시 정리 — 비용 절감
Retain - 온프레미스 유지 (보안·규정 이슈) 단기 이전 불가 시스템 — 재검토 일정 수립
클라우드 비용 모델: On-Demand (사용량 과금, 유연성 최고) | Reserved (약정 할인, 1~3년, 최대 75% 절감) | Spot (잉여 자원 저가, 중단 가능 — 배치 작업용)

학습 확인 퀴즈

Q1. 머신러닝의 지도학습·비지도학습·강화학습 차이를 각각 예시와 함께 설명하시오.

▶ 모범 답안

지도학습: 라벨(정답)이 있는 데이터로 학습. 스팸 메일 필터(스팸/정상 라벨), 신용대출 승인모델(승인/거절 라벨)
비지도학습: 라벨 없이 패턴 스스로 발견. 고객 세분화(유사 고객 군집 발견), 이상거래 탐지(비정상 패턴 찾기)
강화학습: 시행착오+보상/페널티로 최적 행동 학습. AlphaGo(바둑), 로봇 팔 제어, 자율주행 경로 최적화

Q2. 클라우드 마이그레이션 6R 전략 중 Rehost, Refactor, Retire를 각각 설명하고 PM이 전략을 선택하는 기준을 서술하시오.

▶ 모범 답안

Rehost: 코드 변경 없이 그대로 클라우드로 이전(Lift & Shift). 가장 빠르고 리스크 낮음. 마이그레이션 타임라인이 촉박할 때 선택
Refactor: 클라우드 네이티브(MSA, 서버리스)로 완전 재설계. 장기 확장성과 비용 최적화가 목표. 투자 비용이 크고 시간이 오래 걸림
Retire: 사용하지 않는 시스템 폐기. 유지보수 비용 절감

PM 선택 기준: ① 마이그레이션 일정(빠름→Rehost), ② 장기 TCO 목표(절감→Refactor), ③ 사용률(낮음→Retire), ④ 규제·보안(클라우드 불가→Retain)

Q3. PM으로서 AI 프로젝트를 관리할 때 가장 유의해야 할 리스크 3가지를 설명하시오.

▶ 모범 답안

① 데이터 품질 리스크: AI 성능은 학습 데이터 품질에 직결됨. "Garbage In, Garbage Out" — 데이터 정제·레이블링·거버넌스 계획을 범위에 포함해야 함
② 모델 편향(Bias) 및 윤리 리스크: 편향된 데이터로 학습하면 차별적 결과 생성 → 금융·의료·채용 시스템에서 법적 분쟁. 설명 가능성(XAI) 요구사항을 명세해야 함
③ MLOps/모델 드리프트 리스크: 배포 후 데이터 패턴 변화로 모델 성능 저하 (드리프트). 일회성 개발이 아닌 지속적 모니터링·재학습 체계(MLOps) 구축이 필요

유형개념IT 사례
직접비
(Direct)		 프로젝트에
직접 귀속		 개발자 인건비, 외주 용역비, 전용 라이선스
간접비
(Indirect)		 여러 프로젝트

고정비
(Fixed)		 규모 관계없이
일정		 연간 라이선스, PM 급여, 서버 임대
변동비
(Variable)		 작업량에
비례		 시간당 계약 인력, 클라우드 사용료
⚠️ 매몰비용 (Sunk Cost): 이미 지출되어 회수 불가.
"이미 10억을 투자했으니 계속해야 한다" → Sunk Cost Fallacy!
미래 비용 vs 미래 편익만으로 의사결정해야 함.
💡 기회비용 (Opportunity Cost): 선택하지 않은 대안의 가치.
프로젝트 A(NPV 10억) 선택 시, B(NPV 7억)가 기회비용 → A의 순편익 = 3억

원가 산정 4기법 — 비교표

기법접근정확도시간/비용필요 정보적용 시점
유사 산정
(Analogous)		 Top-down ROM
-25%~+75%		 빠름/저렴 과거 유사 프로젝트 실적 착수/초기
모수 산정
(Parametric)		 통계 관계식 예산
-10%~+25%		 중간 단가 통계 (단가×수량) 기획 중반
상향식 산정
(Bottom-up)		 Bottom-up 확정
-5%~+10%		 느림/비쌈 상세 WBS 필수 기획 후반
3점 산정
(Three-point)		 확률 분포 중간~높음 중간 전문가 3점 판단 불확실성 高
PERT (베타 분포) — 원가 산정:
기대 원가 = (O + 4M + P) ÷ 6
SD = (P − O) ÷ 6
O=낙관 · M=최빈 · P=비관

계산 예시

O=600만, M=1000만, P=1800만
기대 = (600+4000+1800)÷6 = 1,067만원
SD = (1800-600)÷6 = 200만원
산정 정확도 등급
ROM: -25%~+75% — 착수 단계, Go/No-Go
예산 산정: -10%~+25% — 기획, 예산 승인
확정 산정: -5%~+10% — 실행 직전, 계약 체결
모수 산정 예시
80개 화면 × 500만/화면 × 복잡도 1.2
= 4.8억원

예산 수립 & 원가 기준선

⏱ 1.5시간

프로젝트 예산 구조 & S-Curve

예산 레이어 구조

🔴 프로젝트 예산 (Project Budget)
= 원가 기준선 + 관리 예비
┣ 관리 예비: Unknown Unknowns, 경영진 승인 필요
🟠 원가 기준선 (Cost Baseline)
= 원가 산정치 합 + 우발 예비
┣ 우발 예비: Known Unknowns, PM 권한, 기준선 포함
🟢 원가 산정치 합
WBS 하위 원가 Bottom-up 합산
💡 원가 기준선은 EVM의 PV(계획가치)의 기반.
시간 배분된 원가 기준선 = S-Curve (누적 PV)

📈 S-Curve (누적 지출 곡선)

EV < PV: 일정 지연 / AC > EV: 예산 초과 (최악)

4교시

EVM 획득가치 관리 — 개요

PV · EV · AC · SV · CV · SPI · CPI — 숫자로 프로젝트 성과 측정

⏱ 1시간

EVM — 핵심 3변수 & 성과 지표

PV
계획가치
Planned Value
지금까지 계획된
작업의 예산
EV
획득가치
Earned Value
지금까지 실제
완료한 작업의 예산 가치
AC
실제원가
Actual Cost
지금까지
실제 지출한 비용
지표공식값 해석의미
SV (일정 편차) EV − PV + 앞섬 / − 지연 일정 성과 편차
CV (원가 편차) EV − AC + 절감 / − 초과 예산 성과 편차
SPI (일정 성과 지수) EV ÷ PV >1 앞섬 / <1 지연 일정 효율성
CPI (원가 성과 지수) EV ÷ AC >1 절감 / <1 초과 원가 효율성
BAC (완료 예산) 원가 기준선 합계 프로젝트 전체 예산 Budget at Completion

예시: PV=5억, EV=4억, AC=6억

SV = 4-5 = -1억 (지연) | CV = 4-6 = -2억 (초과) | SPI = 4/5 = 0.8 (일정 80% 효율) | CPI = 4/6 = 0.67 (원가 67% 효율)
5교시

EVM 심화 — 완료 예측

EAC 3가지 공식 · ETC · VAC · TCPI · 프로젝트 상태 진단

⏱ 2시간

EAC — 완료 시 예상 비용 (3가지 공식)

① 현재 추세 지속

EAC = BAC ÷ CPI

현재 CPI가 앞으로도 유지된다고 가정.
가장 일반적으로 사용.
CPI가 나쁘면 EAC가 크게 증가

② 비전형적 차이

EAC = AC + (BAC − EV)

초기 편차는 일시적이며
나머지는 계획대로 될 것.
단발성 문제 해결 후 정상화 시

③ 복합 지표

EAC = AC + (BAC−EV)
÷ (CPI × SPI)

일정과 원가 모두 고려.
프로젝트가 일정·비용 모두 압박 시.
결과가 가장 비관적

지표공식의미
ETC (잔여 원가 산정치)EAC − AC지금부터 완료까지 필요한 추가 비용
VAC (완료 시 편차)BAC − EAC최종 예산 편차 (음수 = 초과)
TCPI (완료 성과 지수)(BAC−EV) ÷ (BAC−AC)목표 달성에 필요한 앞으로의 효율. >1은 더 잘해야 함을 의미

예시: BAC=20억, PV=10억, EV=8억, AC=12억

CPI = 8/12 = 0.667 | SPI = 8/10 = 0.8
EAC① = 20÷0.667 = 30억 | EAC② = 12+(20-8) = 24억 | EAC③ = 12+12÷(0.667×0.8) = 34.5억
TCPI = (20-8)÷(20-12) = 12÷8 = 1.5 (앞으로 50% 더 효율적으로 해야!)

EVM 상태 진단 — 사분면 해석

SPICPI상태조치
> 1 (앞섬)> 1 (절감) ✅ 최상 현 추세 유지, 공유 및 벤치마킹
> 1 (앞섬)< 1 (초과) ⚠️ 일정 Good
비용 Bad		 비용 원인 분석, 자원 효율성 개선 필요
< 1 (지연)> 1 (절감) ⚠️ 일정 Bad
비용 Good		 일정 단축 기법 (Crashing, Fast-tracking) 적용
< 1 (지연)< 1 (초과) 🔴 최악 스폰서 보고, 범위 조정, 긴급 회의 필요
💡 CPI < 0.8은 위험 신호!
연구에 따르면 CPI가 0.8 이하가 되면
프로젝트 종료까지 회복하는 경우가 거의 없음.
→ 즉각적인 PM 보고 + 스폰서 개입 필요

CPI=1 현재 프로젝트 SPI=0.8, CPI=0.67

학습 확인 퀴즈

Q1. 매몰비용이란 무엇이며, 프로젝트 의사결정에 사용하면 안 되는 이유는?

▶ 모범 답안

매몰비용은 이미 지출되어 어떤 결정을 해도 회수 불가능한 비용입니다.
"이미 10억을 투자했으니 계속해야 한다" → Sunk Cost Fallacy (매몰비용 오류).
합리적 의사결정은 향후 투입할 비용 vs 향후 얻을 편익만 비교해야 합니다.

Q2. BAC=20억, EV=8억, AC=12억, PV=10억일 때 SV, CV, SPI, CPI를 계산하고 상태를 진단하시오.

▶ 모범 답안

SV = EV-PV = 8-10 = -2억 (일정 지연)
CV = EV-AC = 8-12 = -4억 (예산 초과)
SPI = 8/10 = 0.8 (일정 80% 효율 — 지연)
CPI = 8/12 = 0.67 (원가 67% 효율 — 심각한 초과)
→ SPI<1, CPI<1으로 최악의 사분면. CPI<0.8이므로 즉각 스폰서 보고 필요.

Q3. 위 상황(BAC=20억, EV=8억, AC=12억)에서 EAC를 3가지 방법으로 계산하시오.

▶ 모범 답안

①현재 추세 지속: EAC = BAC÷CPI = 20÷0.67 = 약 29.9억
②비전형적 차이: EAC = AC+(BAC-EV) = 12+12 = 24억
③복합(CPI×SPI): EAC = 12+12÷(0.67×0.8) = 12+22.4 = 34.4억
→ 현실적으로 ①가 일반적. 스폰서에게 29~34억 초과 가능성 보고.

Q4. 상향식 산정과 유사 산정의 차이를 정확도 및 적용 시점 관점에서 설명하시오.

▶ 모범 답안

유사 산정: Top-down, 과거 유사 프로젝트 참고, 정확도 낮음(ROM: -25%~+75%), 착수~초기 단계.
상향식 산정: Bottom-up, WBS 작업패키지 개별 산정 후 합산, 가장 정확(-5%~+10%), 기획 후반~실행 직전.
→ 초기에는 빠른 유사 산정으로 의사결정 → 실행 전 상향식으로 정확한 원가 기준선 수립.

## 📚 강의 구성

### 오전 세션 (4시간)

#### 1교시: 자원 관리 개요 (1.5시간)
- **자원 관리의 6개 프로세스**
  1. 자원 관리 계획 수립
  2. 활동 자원 산정
  3. 자원 확보
  4. 팀 개발
  5. 팀 관리
  6. 자원 통제
- **자원의 유형**
  - 팀 자원(Team Resources): 프로젝트 팀원
  - 물적 자원(Physical Resources): 장비, 재료, 시설, 인프라
- **조직 구조와 PM 권한**
  - 기능형(Functional): PM 권한 거의 없음
  - 매트릭스형(Matrix): 약한/균형/강한 매트릭스
  - 프로젝트형(Projectized): PM 권한 최대
  - 하이브리드/복합형

#### 2교시: 자원 관리 계획 및 RACI 차트 (1.5시간)
- **자원 관리 계획서 구성요소**
  - 자원 식별 및 역할 정의
  - 역할과 책임 배분
  - 프로젝트 조직도
  - 자원 확보 방법 (내부 배정, 외부 조달)
  - 교육 계획
  - 팀 개발 전략
- **책임 배분 매트릭스(RAM: Responsibility Assignment Matrix)**
- **RACI 차트**
  - **R(Responsible)**: 실행 책임자 - 작업을 실제 수행
  - **A(Accountable)**: 최종 책임자 - 최종 승인/의사결정 (1명만)
  - **C(Consulted)**: 자문 - 의견을 구하는 대상 (양방향 소통)
  - **I(Informed)**: 통보 - 결과를 알려주는 대상 (단방향 소통)
- **RACI 차트 작성 규칙**
  - 각 활동에 A는 반드시 1명
  - R이 없는 활동은 없어야 함
  - 너무 많은 C/I는 의사결정 지연 유발

#### 3교시: 활동 자원 산정 및 자원 확보 (1.5시간)
- **활동 자원 산정**
  - 각 활동에 필요한 자원의 유형과 수량 결정
  - 자원 분류 체계(RBS: Resource Breakdown Structure)
  - 자원 달력(Resource Calendar): 가용 시간대
- **자원 확보**
  - 사전 배정(Pre-assignment)
  - 협상(Negotiation): 기능 관리자와 협의
  - 가상 팀(Virtual Team) 활용
  - 외부 조달(Outsourcing)
- **다중 프로젝트 환경에서의 자원 경합 관리**

### 오후 세션 (4시간)

#### 4교시: 팀 개발 (1.5시간)
- **터크만 모델(Tuckman's Team Development Model)**
  1. **형성기(Forming)**: 팀원 소개, 역할 파악, 독립적 행동
  2. **폭풍기(Storming)**: 의견 충돌, 갈등 발생, 가장 어려운 단계
  3. **규범기(Norming)**: 규칙 수립, 신뢰 형성, 협업 시작
  4. **성과기(Performing)**: 높은 성과, 자율적 문제 해결
  5. **해산기(Adjourning)**: 프로젝트 종료, 팀 해체
- **팀 개발 기법**
  - 집중 배치(Co-location / War Room)
  - 가상 팀(Virtual Teams)
  - 의사소통 기술
  - 인정과 보상(Recognition & Rewards)
  - 교육 및 훈련
  - 팀 빌딩 활동
- **동기부여 이론**
  - 매슬로 욕구단계설 (생리→안전→소속→존경→자아실현)
  - 허즈버그 2요인이론 (위생요인 vs 동기요인)
  - 맥그리거 X-Y 이론
  - 맥클리랜드 성취동기이론 (성취, 권력, 친화)

#### 5교시: 팀 관리 및 갈등 해결 (1.5시간)
- **팀 관리**
  - 팀 성과 평가
  - 문제 해결 및 의사결정
  - 대인관계 기술
- **갈등 관리**
  - 프로젝트에서 갈등 발생 원인: 일정, 우선순위, 자원, 기술적 의견, 성격
  - **갈등 해결 기법** (Preferred → Least preferred)
    1. **협력/문제해결(Collaborate/Problem Solving)**: Win-Win ★가장 이상적
    2. **타협(Compromise/Reconcile)**: Give & Take
    3. **완화(Smooth/Accommodate)**: 공통점 강조, 차이점 축소
    4. **강요(Force/Direct)**: 한쪽 의견 관철 (Win-Lose)
    5. **회피(Withdraw/Avoid)**: 문제 회피, 임시적 해결
- **갈등은 팀의 책임**: PM이 중재하되 팀원 간 직접 해결을 먼저 시도
- **리더십 스타일**
  - 지시적(Directing) → 코칭(Coaching) → 지원적(Supporting) → 위임(Delegating)
  - 상황적 리더십(Situational Leadership)  - **[심화] 서번트 리더십 (Servant Leadership):** 7판 Team 도메인 핵심, Greenleaf 10원칙, 스크럼마스터 vs PM 비교**
- **[심화] DEI (다양성·형평성·포용성):** 무의식적 편향 5유형, 포용적 팀 구축 전략
#### 6교시: 🔨 실습 - RACI 차트 및 팀 시뮬레이션 (1.5시간)
- **실습 1: RACI 차트 작성**
  - IT 시스템 구축 프로젝트의 주요 활동에 대한 RACI 차트 작성
  - 역할: PM, 개발팀장, 설계자, 테스터, 현업 담당자, 스폰서
- **실습 2: 갈등 해결 시뮬레이션**
  - 가상 시나리오 (일정 갈등, 기술 의견 충돌)
  - 각 갈등 해결 기법 적용 롤플레이

---

## 📝 핵심 키워드

| 키워드 | 설명 |
|--------|------|
| RACI | Responsible, Accountable, Consulted, Informed |
| 터크만 모델 | 형성→폭풍→규범→성과→해산 (팀 발전 5단계) |
| 협력/문제해결 | 가장 이상적인 갈등 해결 기법 (Win-Win) |
| 매트릭스 조직 | PM과 기능관리자가 권한 공유 |
| RBS | 자원 분류 체계 |
| 집중 배치 | Co-location, War Room |
| DEI | 다양성(Diversity)·형평성(Equity)·포용성(Inclusion) 팅의(PM 실무 적용) |
| 서번트 리더십 | 7판 Team 도메인 핵심; 장애 제거·환경 조성·성장 지원 |

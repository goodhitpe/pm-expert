## 📚 강의 구성

### 오전 세션 (4시간)

#### 1교시: 애자일 개요 & 선언문 (1시간)
- **애자일(Agile)이란?**: 불확실성·변화에 적응하는 반복적 개발 철학
- **애자일 선언문(Agile Manifesto) 4가지 핵심 가치**
  - 프로세스·도구 < 개인과 상호작용
  - 포괄적 문서 < 작동하는 소프트웨어
  - 계약 협상 < 고객과의 협력
  - 계획을 따르기 < 변화에 대응
- **12가지 원칙** (조기·지속적 가치 인도, 변경 환영, 짧은 주기 인도 등)
- **워터폴 vs 애자일 비교**
  - 요구사항 고정 여부, 고객 참여 빈도, 인도 방식, 문서화 수준
- **애자일 방법론 종류**: 스크럼, 칸반, XP, 린, SAFe

#### 2교시: 스크럼 프레임워크 (1.5시간)
- **3가지 역할(Roles)**
  - Product Owner(PO): 제품 비전, 백로그 관리, 우선순위 결정
  - Scrum Master(SM): 프로세스 코치, 장애물 제거, 팀 보호
  - 개발 팀: 교차기능·자기조직화 팀 (3–9명)
- **5가지 이벤트(Events)**
  - 스프린트(Sprint): 1–4주 타임박스 반복 주기
  - 스프린트 계획(Sprint Planning): 목표 설정, 백로그 선정, 작업 분해
  - 일일 스크럼(Daily Scrum): 15분, 3가지 질문 (어제/오늘/블로커)
  - 스프린트 리뷰(Sprint Review): 완성 증분 시연, 이해관계자 피드백
  - 스프린트 회고(Sprint Retrospective): Start/Stop/Continue
- **3가지 산출물(Artifacts)**
  - 제품 백로그(Product Backlog): 우선순위화된 요구사항 목록
  - 스프린트 백로그(Sprint Backlog): 선정 항목 + 작업 분해
  - 증분(Increment): DoD 충족, 잠재적 출시 가능
- **사용자 스토리 & INVEST 기준**
- **스토리 포인트 & 플래닝 포커**
- **벨로시티와 번다운 차트**

#### 3교시: 칸반 & 하이브리드 방법론 (1시간)
- **칸반(Kanban)**: 흐름 기반 지속적 인도
  - 칸반 보드 컬럼 구성 (Backlog → To Do → In Progress → Review → Done)
  - WIP(Work In Progress) 제한: 동시 진행 작업 수 제한, 병목 방지
  - 리드타임 & 사이클타임 측정
  - 칸반 vs 스크럼 비교표
- **하이브리드 방법론(Hybrid)**
  - Water-Scrum-Fall 패턴
  - 어떤 상황에서 하이브리드를 선택하나
- **SAFe 소개** (대규모 조직 애자일 확장 프레임워크)
- **PM 관점 애자일 전환 3단계**: 경영진 설득 → 파일럿 → 조직 확산
- **실습**: 제품 백로그 작성 + 스프린트 계획 수립

#### 4교시(오전): 모의 스프린트 (30분)
- 팀별 2주 스프린트 시뮬레이션
  - Sprint Planning → Daily Standup → Sprint Review → Retrospective
- 번다운 차트 실시간 업데이트 실습

### 오후 세션 (4시간)

#### 5교시: Jira 실습 (1.5시간)
- **Jira 핵심 개념**: 프로젝트, 이슈, 워크플로우, 대시보드
- **스크럼 보드 설정**: 스프린트 생성, 백로그 관리, 번다운 차트 자동화
- **칸반 보드 설정**: WIP 제한 설정, 컬럼 커스터마이징
- **JQL(Jira Query Language) 기초**: 이슈 필터링, 빠른 검색
- **보고서 활용**: 번다운, 번업, 속도 차트, 스프린트 보고서
- **실습**: Jira 프로젝트 생성 → 백로그 입력 → 스프린트 시작

#### 6교시: MS Project & Trello 실습 (1.5시간)
- **MS Project 핵심 기능**
  - 간트차트 작성: 작업, 기간, 의존성 설정
  - 자원 배정 및 과부하 확인
  - 크리티컬 패스 시각화
  - 기준선(Baseline) 저장 및 진척도 추적
  - EVM 보고서 생성
- **Trello 핵심 기능**
  - 보드/리스트/카드 구조
  - 체크리스트, 마감일, 담당자 설정
  - Power-Up(자동화, 캘린더, 지라 연동)
  - 소규모 프로젝트·개인 업무 관리에 최적
- **도구 선택 가이드**: 프로젝트 규모·방법론별 적합 도구 비교
  | 상황 | 추천 도구 | 이유 |
  |------|----------|------|
  | 스크럼 팀 (5–50명) | Jira | 스프린트/백로그 통합 |
  | 워터폴·대형 프로젝트 | MS Project | 간트/EVM/자원 관리 |
  | 소규모·개인 업무 | Trello | 단순·직관적 kanban |
  | 하이브리드 | Jira + Confluence | PM + 문서 통합 |

#### 7교시: 종합 실습 — 실전 프로젝트 실습 (1시간)
- **팀별 과제**: 가상 프로젝트를 선택하고 Jira에 다음을 구성
  - 제품 백로그 10개 이상 (사용자 스토리 형식)
  - 2주 스프린트 계획 (스토리 포인트 배분)
  - RACI 차트 연동
  - 번다운 차트 확인
- **발표**: 팀별 5분 스프린트 리뷰 발표

## 📊 핵심 비교 요약

### 워터폴 vs 스크럼 vs 칸반

| 항목 | 워터폴 | 스크럼 | 칸반 |
|------|--------|--------|------|
| 요구사항 | 초기 확정 | 점진적 발견 | 지속 변화 |
| 인도 주기 | 프로젝트 종료 | 스프린트(2주) | 연속적 |
| 계획 단위 | 전체 | 스프린트 | 작업 단위 |
| 변경 대응 | 통제 대상 | 스프린트 경계 | 언제든 가능 |
| 성과 지표 | 마일스톤 | 벨로시티 | 리드타임 |
| 적합 프로젝트 | 요건 명확한 대형 | 불확실·반복 개발 | 운영·유지보수 |

## ✅ 학습 체크리스트

- [ ] 애자일 선언문 4가지 가치를 암기 없이 설명할 수 있다

- [ ] 워터폴/스크럼/칸반 중 어떤 상황에 무엇을 쓸지 판단할 수 있다
- [ ] Jira에서 스크럼 보드와 번다운 차트를 생성할 수 있다
- [ ] MS Project에서 간트차트와 크리티컬 패스를 작성할 수 있다
- [ ] 프로젝트 특성에 맞는 PM 도구를 선택할 수 있다

## 📚 강의 구성

### 오전 세션 (4시간)

#### 1교시: 네트워크 기초 개념 (1.5시간)
- **네트워크(Network)란?**: 장치 간 데이터 전달 체계
- **IP 주소(IP Address)**
  - IPv4 구조: 32비트, 4옥텟 표기 (예: 192.168.1.10)
  - 사설 IP(Private IP) vs 공인 IP(Public IP)
  - NAT(Network Address Translation): 사설↔공인 변환
- **서브넷(Subnet)과 서브넷 마스크**
  - 서브넷 마스크: /24, /16, /8의 의미
  - CIDR 표기법: 192.168.0.0/24 → 호스트 254개
  - 서브넷 분할: 용도별 네트워크 분리 (업무/개발/DMZ)
- **포트(Port)와 프로토콜**
  - 잘 알려진 포트: 80(HTTP), 443(HTTPS), 22(SSH), 3306(MySQL), 5432(PostgreSQL)
  - TCP vs UDP: 신뢰성 vs 속도 트레이드오프
- **PM 실무 적용**: 방화벽 정책 요청서 읽기, 네트워크 다이어그램 해석

#### 2교시: 인터넷 통신 원리 (1.5시간)
- **DNS(Domain Name System)**
  - 도메인 → IP 주소 변환 과정 (재귀 조회 흐름)
  - DNS 레코드 유형: A, CNAME, MX, TXT
  - TTL의 의미와 캐시 영향
  - **PM 관점**: 도메인 이전·CDN 적용 시 DNS 전파 지연(최대 48시간) 일정 반영
- **HTTP vs HTTPS**
  - HTTP 요청·응답 구조 (메서드, 상태 코드)
  - HTTPS: TLS 암호화, 인증서(SSL Certificate) 관리
  - 상태 코드 PM 활용: 200(성공), 301/302(리다이렉트), 404(없음), 500(서버 오류)
- **TCP vs UDP 비교**
  - TCP: 3-way Handshake, 신뢰성 보장, 순서 보장
  - UDP: 속도 우선, 스트리밍·게임·실시간 통신
- **로드밸런서(Load Balancer)**: 트래픽 분산, 고가용성(HA) 확보
- **CDN(Content Delivery Network)**: 정적 자원 엣지 캐싱, 글로벌 서비스 응답 속도 개선

#### 3교시: 네트워크 장비와 아키텍처 (1시간)
- **주요 네트워크 장비**
  - 라우터(Router): 서로 다른 네트워크 간 경로 결정
  - 스위치(Switch): 동일 네트워크 내 장치 연결
  - 방화벽(Firewall): 허용/차단 정책 기반 트래픽 필터링
  - 프록시 서버(Proxy): 중개·캐싱·접근 제어
- **온프레미스 네트워크 구성 패턴**
  - 인터넷 → 방화벽 → DMZ (웹·API 서버) → 내부망 (DB·ERP)
  - VLAN: 물리적 분리 없이 논리적 네트워크 분리
- **WAN vs LAN vs VPN**
  - VPN(Virtual Private Network): 공인 인터넷 위에서 암호화된 사설 연결
  - 사이트 간 VPN(Site-to-Site): 본사↔지사 연결
- **PM 관점**: 네트워크 변경은 변경 관리(CCB) 대상, 장애 발생 시 업무 영향도 평가

### 오후 세션 (4시간)

#### 4교시: 온프레미스 vs 클라우드 네트워크 (1.5시간)
- **왜 클라우드로 가는가?**
  - CapEx → OpEx 전환, 탄력적 확장, 글로벌 인프라
  - 온프레미스 한계: 초기 투자, 용량 예측 실패, 재해 복구 비용
- **클라우드 네트워크 개념 전환**
  - 물리 장비 → 소프트웨어 정의 네트워크(SDN)
  - 데이터센터 리전(Region) + 가용 영역(AZ: Availability Zone)
  - 리전/AZ 선택이 프로젝트 비용·레이턴시·컴플라이언스에 미치는 영향
- **클라우드 주요 사업자 비교**
  - AWS: 가장 성숙, 리전 33개 이상
  - Azure: MS 생태계(AD, Office365) 통합 강점
  - GCP: 빅데이터·AI 워크로드 강점
- **마이그레이션 전략(6R)**
  - Rehost(Lift & Shift), Replatform, Repurchase, Refactor, Retire, Retain
- **PM 관점**: 클라우드 마이그레이션 프로젝트 리스크 — 네트워크 레이턴시, 데이터 전송 비용, 인증·보안 변경

#### 5교시: VPC 핵심 개념 (1.5시간)
- **VPC(Virtual Private Cloud)란?**
  - 클라우드 내 논리적으로 격리된 사설 네트워크 공간
  - 내가 정의하는 IP 범위(CIDR), 서브넷, 라우팅 테이블, 게이트웨이
- **서브넷(Subnet)**
  - 퍼블릭 서브넷(Public Subnet): 인터넷 게이트웨이 연결, 외부 접근 가능
  - 프라이빗 서브넷(Private Subnet): 외부 직접 접근 불가, DB·백엔드 배치
  - AZ별 서브넷 분산: 고가용성(Multi-AZ) 구성
- **게이트웨이와 라우팅**
  - 인터넷 게이트웨이(IGW): VPC ↔ 인터넷
  - NAT 게이트웨이: 프라이빗 서브넷 → 인터넷 (단방향, 아웃바운드만)
  - 라우팅 테이블(Route Table): 트래픽 경로 정의
- **VPC 표준 3계층 아키텍처**
  - Public Tier: ALB(로드밸런서), Bastion Host
  - Private Tier: 앱 서버, API 서버
  - Data Tier: RDS, ElastiCache (가장 격리)
- **VPC Peering & Transit Gateway**: VPC 간 연결, 멀티 VPC 허브 관리

#### 6교시: VPC 보안 & 실습 (1시간)
- **Security Group(보안 그룹)**
  - 인스턴스(EC2) 단위 방화벽, Stateful(응답 자동 허용)
  - 인바운드/아웃바운드 규칙 (프로토콜, 포트, 소스 CIDR)
- **Network ACL(NACL)**
  - 서브넷 단위 방화벽, Stateless(요청·응답 각각 규칙 필요)
  - 허용/차단 번호 규칙(낮은 번호 우선)
- **Security Group vs NACL 비교표**
- **온프레미스 ↔ VPC 연결**
  - AWS Site-to-Site VPN: 암호화 터널, 즉시 사용, 인터넷 의존
  - AWS Direct Connect: 전용 회선, 안정적 대역폭, 도입 시간↑ 비용↑
- **🔨 실습: VPC 아키텍처 다이어그램 설계**
  - 가상 e커머스 시스템의 VPC 구성 설계
  - Public/Private/Data Subnet 배치
  - Security Group 규칙 정의 (웹서버 80/443, DB 3306 등)
  - 팀별 발표 및 피드백

## 📝 핵심 키워드

| 키워드 | 설명 |
|--------|------|
| CIDR | IP 주소 범위 표기법, /24 = 254 호스트 |
| NAT | 사설 IP ↔ 공인 IP 변환 (또는 Private Subnet 아웃바운드) |
| DNS | 도메인 → IP 변환, TTL 전파 지연 일정 고려 |
| VPC | 클라우드 내 논리 격리 사설 네트워크 |
| 퍼블릭/프라이빗 서브넷 | 인터넷 접근 가능 여부로 구분, 계층별 보안 분리 |
| IGW / NAT GW | 인터넷 게이트웨이(양방향) vs NAT 게이트웨이(단방향) |
| Security Group | 인스턴스 단위·Stateful 방화벽 |
| NACL | 서브넷 단위·Stateless 방화벽 |
| Direct Connect | VPN 대비 안정적인 전용 회선 클라우드 연결 |
| Multi-AZ | 가용 영역 분산 배치로 고가용성 확보 |

# Day 2: 프로젝트 통합 관리 (Project Integration Management)

## 📋 강의 개요

| 항목 | 내용 |
|------|------|
| **과목명** | 프로젝트 통합 관리 |
| **교육시간** | 8시간 (오전 4시간 / 오후 4시간) |
| **학습목표** | 프로젝트 통합 관리의 7개 프로세스를 이해하고 프로젝트 헌장을 작성할 수 있다 |
| **선수학습** | Day 1 오리엔테이션 |

---

## 🎯 학습 목표

1. 통합 관리가 프로젝트 관리에서 가장 핵심적인 지식영역인 이유를 설명할 수 있다
2. 프로젝트 헌장의 구성요소를 이해하고 작성할 수 있다
3. 프로젝트 관리 계획서의 구조와 역할을 설명할 수 있다
4. 통합 변경 통제 프로세스를 이해하고 적용할 수 있다

---

## 📚 강의 구성

### 오전 세션 (4시간)

#### 1교시: 통합 관리 개요 (1.5시간)
- **통합 관리란?**: 프로젝트 관리의 모든 지식영역을 조율하고 통합하는 역할
- **통합 관리의 중요성**: PM만이 수행하는 유일한 지식영역
- **7개 프로세스 맵**
  1. 프로젝트 헌장 개발 (착수)
  2. 프로젝트 관리 계획서 수립 (기획)
  3. 프로젝트 작업 지시 및 관리 (실행)
  4. 프로젝트 지식 관리 (실행)
  5. 프로젝트 작업 감시 및 통제 (감시 및 통제)
  6. 통합 변경 통제 수행 (감시 및 통제)
  7. 프로젝트 또는 단계 종료 (종료)

#### 2교시: 프로젝트 헌장 개발 (1.5시간)
- **프로젝트 헌장(Project Charter)의 정의**: 프로젝트의 존재를 공식적으로 인가하는 문서
- **헌장의 주요 구성요소**
  - 프로젝트 목적 및 정당성(Business Case)
  - 측정 가능한 프로젝트 목표 및 성공 기준
  - 상위 수준의 요구사항 및 프로젝트 설명
  - 상위 수준 리스크
  - 요약 마일스톤 일정
  - 요약 예산
  - 프로젝트 승인 요구사항
  - 임명된 PM 및 권한 수준
  - 프로젝트 스폰서
- **프로젝트 스폰서(Sponsor)의 역할**
  - 헌장 승인 권한
  - 자원 확보 지원
  - 주요 의사결정 에스컬레이션

#### 3교시: 프로젝트 관리 계획서 수립 (1.5시간)
- **프로젝트 관리 계획서**: 프로젝트 실행, 감시 및 통제, 종료 방법을 정의하는 문서
- **보조 관리 계획서 (Subsidiary Plans)**
  - 범위 관리 계획서
  - 일정 관리 계획서
  - 원가 관리 계획서
  - 품질 관리 계획서
  - 자원 관리 계획서
  - 의사소통 관리 계획서
  - 리스크 관리 계획서
  - 조달 관리 계획서
  - 이해관계자 참여 계획서
- **기준선(Baseline)**: 범위 기준선, 일정 기준선, 원가 기준선
- **프로젝트 생애주기(Life Cycle) 정의**

### 오후 세션 (4시간)

#### 4교시: 프로젝트 작업 지시 및 지식 관리 (1.5시간)
- **작업 지시 및 관리**
  - 인도물(Deliverables) 생산
  - 변경요청 관리
  - 작업 성과 데이터 수집
- **프로젝트 지식 관리**
  - 명시적 지식(Explicit Knowledge) vs 암묵적 지식(Tacit Knowledge)
  - 교훈 등록부(Lessons Learned Register)
  - 지식 저장소(Knowledge Repository)

#### 5교시: 감시/통제 및 통합 변경 통제 (1.5시간)
- **프로젝트 작업 감시 및 통제**
  - 작업 성과 보고서
  - 계획 대비 실적 비교
  - 편차(Variance) 분석
- **통합 변경 통제 수행**
  - 변경 통제 위원회(CCB: Change Control Board)
  - 변경 요청 프로세스: 제출 → 검토 → 승인/반려 → 반영
  - 형상 관리(Configuration Management)
  - 승인된 변경 요청 vs 반려된 변경 요청
- **실무에서의 변경 관리 주의사항**
  - Gold Plating(과도한 기능 추가) 방지
  - 변경의 영향 분석 중요성

#### 6교시: 프로젝트/단계 종료 + 실습 (1.5시간)
- **프로젝트 또는 단계 종료**
  - 최종 인도물 인수
  - 교훈 수집 및 정리
  - 계약 종료
  - 조직 프로세스 자산 업데이트
  - 최종 보고서 작성
- **조기 종료 사유**
  - 프로젝트 취소, 자원 부족, 사업 환경 변화
- **🔨 실습: 프로젝트 헌장 작성**
  - 가상 IT 프로젝트 시나리오 제공
  - 팀별로 프로젝트 헌장 작성
  - 발표 및 피드백

## 📝 핵심 키워드

| 키워드 | 설명 |
|--------|------|
| 프로젝트 헌장 | 프로젝트를 공식 인가하는 문서 |
| CCB | 변경 통제 위원회 (Change Control Board) |
| 기준선 | 승인된 범위/일정/원가 계획 (비교 기준) |
| 교훈 등록부 | 프로젝트 중 획득한 교훈을 기록하는 문서 |
| 형상 관리 | 제품의 기능적/물리적 특성을 문서화하고 통제 |
| Gold Plating | 승인되지 않은 추가 작업/기능 제공 |

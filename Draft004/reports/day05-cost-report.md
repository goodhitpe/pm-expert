## 4. 보완 자료 (슬라이드 미포함 핵심 내용)

### 4-1. EVM 상세 계산 예제 — 클라우드 전환 프로젝트 완전 풀이

**프로젝트 조건**: 총 예산(BAC) = ₩1,000만, 10주 완료 계획

| 지표 | 값 | 계산 | 해석 |
|------|-----|------|------|
| PV (5주차) | ₩500만 | BAC × 50% | 이 시점 계획값 |
| EV | ₩400만 | 실제 완료분 가치 | 계획 대비 80% 진도 |
| AC | ₩480만 | 실제 투입 비용 | |
| SV = EV-PV | **-₩100만** | 400-500 | 일정 지연 (음수=지연) |
| CV = EV-AC | **-₩80만** | 400-480 | 원가 초과 (음수=초과) |
| SPI = EV/PV | **0.80** | 400/500 | 계획의 80% 속도 |
| CPI = EV/AC | **0.83** | 400/480 | ₩1 투입 시 ₩0.83 획득 |
| EAC (CPI 기반) | **₩1,205만** | BAC/CPI = 1000/0.83 | 최종 예상 원가 |
| ETC = EAC-AC | **₩725만** | 1205-480 | 잔여 작업 예상 비용 |
| VAC = BAC-EAC | **-₩205만** | 1000-1205 | 예산 초과 예상액 |
| TCPI (BAC 기준) | **1.30** | (BAC-EV)/(BAC-AC) = (1000-400)/(1000-480) | 남은 작업 효율 목표 |

> **TCPI 해석**: TCPI=1.30 → 남은 예산으로 목표 달성하려면 현재보다 30% 더 효율적이어야 함. CPI=0.83보다 TCPI=1.30이 훨씬 높으면 목표 달성 사실상 불가능 → EAC 재산정 필요.

### 4-2. TCPI vs CPI 비교 해석 기준

| CPI | TCPI (BAC 기준) | 의미 | PM 액션 |
|-----|----------------|------|--------|
| > 1.1 | < 1.0 | 예산 내 완료 가능, 여유 있음 | 현상 유지 |
| 1.0 | 1.0 | 완벽한 효율로 달성 가능 | 현상 유지 |
| 0.9~1.0 | 1.0~1.1 | 약간 노력 필요 | 효율 개선 계획 |
| < 0.9 | > 1.1 | 재산정 필요 가능성 높음 | EAC 재계산 |
| < 0.8 | > 1.25 | BAC 달성 사실상 불가능 | 스폰서 알림, EAC 공식 재산정 |

### 4-4. 생애주기 원가 (LCC) — SaaS vs On-Premise 5년 비교

| 항목 | SaaS (클라우드) | On-Premise | 비고 |
|------|--------------|-----------|------|
| 초기 투자 | ₩500만 (구축 없음) | ₩8,000만 (서버+SW) | CAPEX vs OPEX |
| 연간 라이선스 | ₩1,200만/년 | ₩200만/년 (유지보수) | |

### 4-5. 원가 산정 정확도 등급 비교

| 산정 등급 | 시점 | 정확도 범위 | 활용 |
|---------|------|-----------|------|
| ROM (Rough Order of Magnitude) | 개시 전·헌장 단계 | -25% ~ +75% | 타당성 검토 |
| Budget Estimate | 기획 단계 | -10% ~ +25% | 예산 승인 요청 |
| Definitive Estimate | 설계 완료 후 | -5% ~ +10% | 계약·원가 기준선 |

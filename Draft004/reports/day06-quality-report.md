## 4. 보완 자료 (슬라이드 미포함 핵심 내용)

### 4-1. 히스토그램 패턴 3종 해석

| 패턴 | 모양 | 의미 | PM 조치 |
|------|------|------|--------|
| **정규분포 (Bell Curve)** | 중앙 대칭 종 모양 | 공정이 안정적, 자연적 변동 | 현상 유지 |
| **편중 분포 (Skewed)** | 한쪽으로 치우침 | 공정에 편향 존재 (과도한 사양, 절삭 등) | 원인 조사 |
| **이봉 분포 (Bimodal)** | 두 개의 봉우리 | 두 가지 다른 프로세스/그룹 혼재 | 데이터 분리 분석 |
| **이상값 (Outlier)** | 고립된 막대 | 규격 이탈, 측정 오류 | 즉시 조사·제거 |

### 4-2. 산포도 상관관계 유형

| 유형 | 설명 | 상관계수(r) | IT 예시 |
|------|------|-----------|--------|
| **강한 양의 상관** | X 증가 → Y 증가 (선형) | r ≈ +0.9 이상 | 테스트 케이스 수 ↑ → 결함 발견 수 ↑ |
| **약한 양의 상관** | X 증가 → Y 약간 증가 | r = +0.3~0.7 | 코드 라인 수 ↑ → 개발 기간 약간 ↑ |
| **무상관** | X와 Y 관계 없음 | r ≈ 0 | 개발자 나이 vs 결함 수 |
| **강한 음의 상관** | X 증가 → Y 감소 | r ≈ -0.9 이상 | 코드 리뷰 횟수 ↑ → 결함 수 ↓ |
| **비선형 관계** | 곡선형 관계 | 산포도로만 파악 가능 | 테스트 시간 vs 발견 결함 (초기 급증 후 감소) |

> **Rule of Seven 연계**: 관리도에서 7개 연속 같은 방향 = 비무작위 패턴, 산포도에서 7개 연속 같은 방향 = 유의미한 상관 신호.

### 4-3. 린 7대 낭비 (Muda) — IT 프로젝트 적용

| 낭비 유형 | 제조업 원개념 | IT 프로젝트 적용 예시 |
|---------|-----------|------------------|
| **재고 (Inventory)** | 미사용 원자재 | 우선순위 없는 백로그 누적, 완료 안 된 기능 |
| **대기 (Waiting)** | 공정 간 대기 | 코드 리뷰 대기, 승인 지연, QA 큐 대기 |
| **운반 (Transportation)** | 불필요한 이동 | 불필요한 파일 이동, 시스템 간 데이터 재입력 |
| **동작 (Motion)** | 작업자 불필요 동작 | 비효율 도구 전환, 반복 클릭, 수동 리포트 |
| **과잉 생산 (Overproduction)** | 미리 만드는 것 | 사용하지 않을 기능 개발, 불필요 문서 |
| **과잉 처리 (Over-Processing)** | 필요 이상 가공 | 사용 않는 API 필드, 과도한 문서화 |
| **결함 (Defects)** | 불량품 | 버그, 재작업, 테스트 재수행 |

> 린의 제8낭비 (일부 모델): **인재(Talent) 낭비** — 직원 역량을 최대 활용하지 못함.

### 4-4. CMMI 5단계별 핵심 프로세스 영역

| 단계 | 명칭 | 핵심 특성 | 핵심 프로세스 영역 |
|------|------|---------|----------------|
| **1 (Initial)** | 혼돈 | 프로세스 없음, 개인 영웅에 의존 | - |
| **2 (Managed)** | 관리됨 | 프로젝트 수준 기본 프로세스 존재 | 요구사항 관리, CM, QA, 프로젝트 계획 |
| **3 (Defined)** | 정의됨 | 조직 표준 프로세스 정립 | 조직 프로세스 정의, 교육훈련, IPM |
| **4 (Quantitatively Managed)** | 정량적 관리 | 측정·통계 기반 관리 | 정량적 프로세스 관리, SW 품질 관리 |
| **5 (Optimizing)** | 최적화 | 지속적 프로세스 개선 | 혁신·배포, 원인 분석·해결 |

> 국내 현황: 대기업 SI 업체 대부분 CMMI 3~4 수준. 레벨 5 기업은 IBM, 인포시스 등 글로벌 대형업체.

### 4-5. 6시그마 DMAIC 단계별 핵심 활동

| 단계 | 목적 | 주요 도구 | 산출물 |
|------|------|---------|------|
| **Define** | 문제·목표 정의 | SIPOC, 고객 VOC, Project Charter | 식스시그마 헌장 |
| **Measure** | 현 수준 측정 | 관리도, 시그마 수준 계산, MSA | 베이스라인 시그마 |
| **Analyze** | 근본 원인 분석 | 인과도(Fishbone), 파레토, 가설검정 | 핵심 X 인자 목록 |
| **Improve** | 개선안 적용 | DOE, Poka-yoke, 파일럿 | 개선된 프로세스 |
| **Control** | 개선 지속 | 관리도, 표준화, SOP | 통제 계획서 |

> **시그마 수준**: 6시그마 = 100만 기회당 3.4개 불량. 현실적 목표는 4~5시그마 (100만 당 233~6,210개).

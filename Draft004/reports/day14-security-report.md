## 시험 포인트 (PMP/자격증 관련)

### 🔴 고빈도 출제 항목

1. **CIA Triad 3요소 + 구현 기술**
   - 기밀성 → 암호화, 접근통제
   - 무결성 → 해시, 디지털 서명
   - 가용성 → 이중화, 백업, DR

2. **RPO vs RTO 구분**
   - RPO = 데이터 손실 허용 시점 (백업 주기)
   - RTO = 서비스 복구 목표 시간

3. **대칭키 vs 비대칭키**
   - 대칭키(AES): 빠름, 키 배포 문제
   - 비대칭키(RSA): 느림, 키 배포 안전

4. **접근 통제 3방식**
   - DAC: 소유자 재량
   - MAC: 시스템/정책 결정
   - RBAC: 역할(직무)별 권한 — 실무 일반적

5. **최소 권한 원칙 & 직무 분리**
   - 두 원칙은 항상 함께 출제됨

6. **보안 사고 대응 6단계 순서**
   - 준비 → 식별 → 억제 → 제거 → 복구 → 교훈

7. **MFA 3요소**
   - 지식(패스워드) + 소유(OTP) + 생체(지문) — 2개 이상 결합

## 보완 자료 (슬라이드 미포함 핵심 내용)

### B-1. NIST CSF (Cybersecurity Framework) 5대 기능 상세

| 기능 | 핵심 활동 | IT PM 적용 |
|------|---------|-----------|
| **Identify (식별)** | 자산 목록, 리스크 평가, 거버넌스 정책 수립 | 프로젝트 착수 시 보안 자산 목록화 |
| **Protect (보호)** | 접근 통제, 데이터 암호화, 보안 교육, 방화벽 | 구현 단계 보안 요구사항 내재화 |
| **Detect (탐지)** | 이상 탐지(SIEM), 로그 모니터링, IDS/IPS | 운영 단계 지속 모니터링 |
| **Respond (대응)** | 사고 대응 계획, 봉쇄, 이해관계자 통보 | BCP/DRP 사전 준비 |
| **Recover (복구)** | 시스템 복원, 사후 분석, 개선 조치 | RTO/RPO 기준 준수 |

> **버전 2.0 (2024)**: Govern(거버넌스) 기능 추가 → 6대 기능으로 확장

### B-2. 보안 사고 대응 6단계 — PICERL 모델

| 단계 | 영문 | 핵심 활동 |
|------|------|---------|
| 1. 준비 | Preparation | 대응팀 구성, 도구 확보, 플레이북 작성 |
| 2. 식별 | Identification | 사고 탐지, 분류(Triage), 범위 파악 |
| 3. 격리 | Containment | 피해 확산 차단 (단기: 네트워크 차단, 장기: 패치) |
| 4. 박멸 | Eradication | 악성코드 제거, 취약점 패치, 계정 리셋 |
| 5. 복구 | Recovery | 시스템 복원, 모니터링 강화, 정상 운영 재개 |
| 6. 사후 분석 | Lessons Learned | 원인 분석, 재발 방지 조치, 문서화 |

### B-3. 침투 테스트 (Penetration Testing) 3단계 + 유형

**3단계 프로세스**:
```
계획(Planning) → 수행(Execution) → 보고(Reporting)
```

| 단계 | 세부 활동 |
|------|---------|
| 계획 | 범위 정의, 법적 승인(Rules of Engagement), 대상 환경 파악 |
| 수행 | 정보 수집 → 취약점 스캔 → 익스플로잇 → 포스트 익스플로잇 |
| 보고 | 발견사항 분류(CVSS 점수), 재현 절차, 권고 조치 |

**침투 테스트 유형**:

| 유형 | 정보 수준 | 특징 |
|------|---------|------|
| Black Box | 없음 | 실제 외부 공격자 시뮬레이션 |
| White Box | 전체 접근 | 코드·아키텍처 검토 포함, 가장 철저 |
| Gray Box | 부분 (일반 사용자 권한) | 내부자 위협 시뮬레이션 |

### B-4. 주요 보안 규정·표준 비교

| 표준/규정 | 적용 대상 | 핵심 요구사항 |

| HIPAA | 미국 의료 정보 | PHI 보호, BAA 계약 |

## 시험 포인트 (PMP/자격증 관련)

### 🔴 고빈도 출제 항목

1. **4사분면 전략 매칭**
   - "규제 기관(고권력·저관심)" → Keep Satisfied
   - "현장 사용자(저권력·고관심)" → Keep Informed
   - 오답 함정: "중요하니까 긴밀관리" → 권력이 기준

2. **참여도 매트릭스 C≠D 해석**
   - C = 현재(Current), D = 희망(Desired)
   - C < D = 참여도 높여야 함 → 전략 필요
   - C = D → 이상적 상태

3. **현저성 모델 3요소**
   - 권력(Power) + 긴급성(Urgency) + 적법성(Legitimacy)
   - 3개 모두 보유 = 정의적/Definitive 이해관계자 (최우선)
   - 2개 보유 = 예측적 이해관계자
   - 1개 보유 = 잠재적 이해관계자

4. **13.1 vs 13.4 구분**
   - 13.1 식별 = 처음 등록부 작성 (프로젝트 초기)
   - 13.4 감시 = 참여도 변화 추적 (프로젝트 전생애)

5. **Champion 정의**
   - PM이 직접 접근 어려운 영역에서 프로젝트를 대신 옹호
   - Leading 단계(주도)의 이해관계자가 Champion 역할 수행

### B-2. OCM (조직 변화 관리) — PM 관점 필수 이해

**Kotter 8단계 변화 모델**:

| 단계 | 내용 | PM 적용 |
|------|------|--------|
| 1. 긴박감 창출 | "왜 변해야 하는가?" 공유 | 프로젝트 킥오프 시 WHY 강조 |
| 2. 연대 구축 | 변화 주도 팀 형성 | 이해관계자 Champion 확보 |
| 3. 비전 수립 | 변화 후 모습 구체화 | 프로젝트 목표·편익 명확화 |
| 4. 소통 | 비전 지속 공유 | 의사소통 계획서 실행 |
| 5. 장애 제거 | 저항 해소 | 거부자(Resistant) 전략 수립 |
| 6. 단기 성과 | 빠른 승리 보여주기 | 파일럿, 증분 릴리즈 |
| 7. 지속 강화 | 변화 문화 정착 | 교육, DoD(완료 정의) 표준화 |
| 8. 내재화 | 새 방식을 문화로 | 거버넌스, PM오피스 정착 |

### B-3. 이해관계자 실제 참여 전략 사례 — IT ERP 도입

|---------|---------|--------|--------|------|

| 현업 부서장 | 高권력·高관심 | Resistant | Supportive | 1:1 미팅, 맞춤 데모, Champion 육성 |
| IT 팀장 | 中권력·高관심 | Neutral | Leading | 기술 의사결정 참여 기회 부여 |
| 일반 사용자 | 低권력·高관심 | Unaware | Supportive | 사용자 교육, FAQ 포털 제공 |
| 감사 부서 | 中권력·低관심 | Unaware | Neutral | 규정 준수 보고서 정기 제공 |

### B-4. 권력-영향력 (Power-Influence) 매트릭스

**Power-Interest Grid의 확장 모델**:

| 구분 | 고영향력 | 저영향력 |
|------|--------|---------|
| **고권력** | Manage Closely (최우선 관리) | Keep Satisfied (만족 유지) |
| **저권력** | Keep Informed (정보 공유) | Monitor (모니터) |

> P-I Grid와 동일하나 "영향력"은 의사결정 능력 + 실질적 프로젝트 영향력(사용자 저항 등)을 포함. 권력이 낮아도 영향력이 높은 이해관계자(핵심 사용자 집단)는 과소평가 금지.

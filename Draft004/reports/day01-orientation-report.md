## 7. 보완 자료 (슬라이드 미포함 핵심 내용)

### 7-1. 일정 단축 3가지 전략

| 전략 | 설명 | 비용 증가 | 리스크 | 적합 상황 |
|------|------|---------|--------|---------|
| **Crashing** | 추가 자원 투입으로 활동 기간 단축 | 높음 | 중간 (품질 위험) | 주공정 단일 활동 |
| **Fast-Tracking** | 순차 활동을 병렬 진행 | 낮음 | 높음 (재작업) | 설계-개발 병렬 |
| **범위 축소** | 인도물·기능 일부 제거 | 없음 | 낮음 | 마감 불변 시 |

> PMBOK 원칙: Fast-tracking은 추가 비용 없이 일정 단축 가능하나 재작업 리스크가 가장 높다.

### 7-3. 포트폴리오 → 프로그램 → 프로젝트 계층 구조

```
조직 전략 (Strategic Goals)
       │
  ┌────┴─────┐
  │  포트폴리오  │  ← 전략 정렬, 투자 최적화 (예: 디지털 전환 포트폴리오)
  └────┬─────┘
       │
  ┌────┴─────┐
  │  프로그램   │  ← 상호 연관 프로젝트 집합, 시너지 (예: 고객 경험 혁신 프로그램)
  └────┬─────┘
       │
  ┌────┼────┐
  │    │    │
프로젝트A B  C     ← 개별 한시적 작업 (ERP / 모바일앱 / 인프라)
```

| 계층 | 초점 | 관리 주체 | 성공 기준 |
|------|------|---------|---------|
| 포트폴리오 | 전략적 가치·투자 균형 | CPO / PMO | ROI·전략 정렬 |
| 프로그램 | 편익(Benefits) 실현 | 프로그램 관리자 | 편익 지속성 |
| 프로젝트 | Triple Constraint | PM | 범위·일정·원가 |

| 선제조건 | 편익 실현 조건 | 직원 교육 완료, 기존 시스템 마이그레이션 |

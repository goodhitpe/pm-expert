## 보완 자료 (슬라이드 미포함 핵심 내용)

### B-1. 문화 차이에 따른 소통 전략 — 고맥락 vs 저맥락

| 구분 | 고맥락 문화 (High-Context) | 저맥락 문화 (Low-Context) |
|------|--------------------------|------------------------|
| 대표 국가 | 한국, 일본, 중국, 아랍권 | 미국, 독일, 북유럽 |
| 소통 방식 | 암묵적, 비언어 중시, 간접적 | 명시적, 직접적, 문서 중심 |
| 의사결정 | 관계·신뢰 기반 | 데이터·논리 기반 |
| 침묵의 의미 | 동의 또는 숙고 | 불만 또는 이해 부족 |
| PM 적용 | 비공식 채널(점심, 복도) 활용 | 공식 회의·이메일 문서화 강조 |

> **다국적 프로젝트 실패 원인 Top 3 (PMI 2019)**: ①언어 장벽, ②문화적 기대 불일치, ③시간대 차이

### B-2. 번다운 차트 (Burndown Chart) 개념

```
스토리 포인트
100 |*
 80 |  *  ← 이상선 (Ideal Burndown)
 60 |      *  *
 40 |           *   *  ← 실제 진행
 20 |                 *
  0 +──────────────────→ 스프린트 기간 (예: 10일)
```

| 지표 | 의미 |
|------|------|
| 이상선 아래 | 계획보다 빠른 진행 (긍정) |
| 이상선 위 | 계획 지연 (리스크 신호) |
| 수평 구간 | 작업 완료 없음 → 블로커 확인 필요 |
| 급격한 하락 | 과소 산정 SP 또는 범위 축소 의심 |

---

### B-3. 소통 기술 선택 기준 — 긴급성·민감도 매트릭스

| | 긴급 | 비긴급 |
|------|------|------|
| **민감 (기밀·갈등)** | 대면 회의 / 화상통화 | 1:1 미팅, 서면 서신 |
| **비민감 (일반 공유)** | 전화 / 즉각 메시지 | 이메일, 포털 게시 |

> **Push vs Pull 선택 기준**: 수신자가 정보를 찾기 어려운 경우 → Push. 대량 정보를 수시로 참조 → Pull (위키, 대시보드).

### B-4. 의사소통 채널 수 공식 — 실전 계산

| 팀원 수 (n) | 채널 수 n(n-1)/2 | 관리 복잡도 |
|-----------|----------------|-----------|
| 5 | 10 | 낮음 |
| 10 | 45 | 중간 |
| 20 | 190 | 높음 |
| 50 | 1,225 | 매우 높음 |

> PMP 단골 문제: "팀원이 5명에서 7명으로 늘어났을 때 추가 채널 수는?" → 10 → 21 = **+11개** 증가.
